# CHRO Heartbeat

## 每次心跳執行

- 無日常心跳任務（CHRO 以週度審核為主，由 cron 觸發）

## 週度組織健康巡視（每週一 08:00 由 cron 觸發）

執行以下 6 步驟巡視流程：

### 步驟一：讀取各 Agent 日誌
- 讀取各 Agent 過去一週的 memory/ 日誌檔案
- 重點關注：異常事件、反覆出現的問題、任務失敗記錄

### 步驟二：統計指標
- 各 Agent 的任務完成率
- 平均回應時間
- Token 消耗（向 CFO 請求數據）

### 步驟三：能力缺口分析
- 哪些任務反覆失敗？根因是什麼？
- 哪些 Skill 被頻繁調用？是否需要優化？
- 是否有任務類型持續增加但無專責 Agent？

### 步驟四：提案
- 發現能力缺口 → 撰寫新 Skill 開發建議 → 交 CEO 審批
- 發現冗餘 → 建議合併或調整 Agent 職責

### 步驟五：模型適配評估
- 檢查各 Agent 的任務複雜度是否匹配當前模型級別
- 若發現不匹配 → 產出模型升降級建議（參閱下方模型評估流程）

### 步驟六：產出報告
- 生成「組織健康週報」精煉摘要
- 透過 sessions_send 發送給 CEO
- CEO 決定是否納入晨間簡報或直接轉呈董事長
