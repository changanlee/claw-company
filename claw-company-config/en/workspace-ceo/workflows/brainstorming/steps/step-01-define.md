# Step 1: Topic Definition

**Progress: Step 1 / 7**

## Goal

Clearly define the scope of the brainstorming topic and classify it to determine the convergence strategy.

## Execution Rules

- 📖 Read the entire step file before taking any action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip steps

## Reference Principle

Read `{{INSTALL_DIR}}/shared/principles/brainstorming.md` — clarify the problem before acting, explore multiple possibilities before converging.

## Instructions

### 1. Topic Definition

Confirm with the Chairman:

- The core question of the topic
- Expected output (decision direction / creative proposals / problem solution)
- Scope boundaries (what is out of scope)

### 2. Topic Classification

Based on the nature of the topic, determine its type and inform the Chairman:

| Type | Characteristics | Default Convergence Method |
|------|-----------------|---------------------------|
| **Decision** | Architecture decisions, option evaluation, problem diagnosis | PM3 (default) |
| **Creative** | Ideation, naming, marketing, open exploration | brain-methods diverge first, PM3 Phase 3 converge |
| **Hybrid** | Strategy + creativity, cross-domain | PM3 skeleton + brain-methods interspersed |

The Chairman can override the auto-classification (e.g., "use PM3 this time" or "just diverge for now").

### 3. Activate Facilitator Mode

Confirm with the Chairman:

> Topic: [subject]
> Classification: [Decision / Creative / Hybrid]
> Convergence method: [PM3 / Diverge first / Hybrid]
>
> I will facilitate this brainstorming session. If we need a specific executive's or engineer's expertise at any point, I will spawn them in real time.
>
> Ready to begin?

- [ ] Topic type classified
