# Step 1: Justify Need

**Progress: Step 1 of 5**

## Goal

Detect signals for a new role and justify its necessity.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip steps

## Instructions

### 1. Detect Trigger Signals

Check the following indicators:

- **Task Accumulation**: A certain type of task keeps increasing but no dedicated Agent handles it
- **Agent Overload**: An Agent's task volume or Token consumption is persistently high
- **Responsibility Gap**: Recurring capability vacuum in org health reports
- **Efficiency Bottleneck**: A certain type of task consistently exceeds time targets
- **Chairman Directive**: Chairman explicitly requests adding a new capability

### 2. Data Support

Collect quantitative data:

- Weekly/monthly trend of related task volume
- Which existing Agents are handling these tasks (and their burden)
- Token consumption data (request from CFO)
- Average completion time for related tasks

### 3. Necessity Justification

Answer the following questions:

1. **Can existing Agents absorb it?** — Solve via responsibility adjustment
2. **Can it be solved through workflow optimization?** — No new Agent needed
3. **Expected benefits of a new Agent?** — Quantify improvement magnitude
4. **Cost impact?** — Estimated monthly Token budget for new Agent

### 4. Conclusion

- **Recommend creating**: Clear signals, data-supported, existing solutions insufficient
- **Recommend deferring**: Signals still weak, try other solutions first
- **Do not recommend**: Existing architecture can meet the need

If conclusion is "do not recommend" or "defer", record the reason and end the workflow.

## Completion Criteria

- [ ] Trigger signals detected and recorded
- [ ] Quantitative data collected
- [ ] Necessity justification completed
- [ ] Conclusion clearly stated
