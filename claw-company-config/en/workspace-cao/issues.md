## Format Specification

Each issue record format:

### [Issue ID] Issue Title
- **Discovery Date:** YYYY-MM-DD
- **Severity:** P0 (Critical) / P1 (High) / P2 (Medium) / P3 (Low)
- **Involved Agent:** CEO / CFO / CIO / COO / CTO / CHRO
- **Problem Description:** Brief description of the discovered issue
- **Root Cause Analysis:** (To be added during remediation phase)
- **Remediation Measures:** (To be added during remediation phase)
- **Verification Result:** (To be added during verification phase)
- **Closure Date:** (To be added upon closure)

## Maintenance Rules

- Open an issue immediately upon discovering a problem
- Archive issues closed for more than 30 days to memory/ logs
- Review the progress of all unresolved issues during the weekly security scan
