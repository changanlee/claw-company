# Step 1: Understand Requirement Scope

**Progress: Step 1 of 4** — Next: Deep-dive into code impact

## Goal

Understand the full scope of the requirement and confirm it's suitable for the quick spec process.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions

### 1. Receive Requirement Description

Receive the requirement description provided by CTO at spawn time. Understand the feature goal.

### 2. Quick Code Scan

Quickly scan related code to assess impact scope:

- Identify related modules and files
- Preliminary assessment of modification scope
- If scope exceeds "small feature" → recommend CTO use the full process

### 3. Distill Requirements

Organize into a structured description:

- **Title**: One-line feature name
- **Problem description**: What problem to solve
- **Solution**: Expected approach
- **In scope**: What will be done
- **Out of scope**: What will not be done

### 4. Create WIP Spec File

Create an initial WIP (Work In Progress) spec file recording the above information.

## Completion Criteria

- ✅ Requirement understood and structured
- ✅ Impact scope preliminarily confirmed
- ✅ WIP spec file created

## Next Step

After confirming completion criteria are met, read and follow: `./step-02-investigate.md`
