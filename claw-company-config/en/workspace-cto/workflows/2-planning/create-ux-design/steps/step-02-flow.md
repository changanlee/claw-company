# Step 2: User Journeys and Information Architecture

**Progress: Step 2 of 4** — Next: UI Component Specs

## Objective

Design user journey maps and information architecture, defining core flows and page structure.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions (sequential, no skipping)

### 1. User journey design

Design journey maps for each primary user role:

- **Entry points**: How users discover/enter the product
- **Core flows**: Step-by-step process to complete main tasks
- **Decision points**: Nodes where users need to make choices
- **Exit points**: Task completion or departure nodes
- **Emotional curve**: User feelings at each stage (anticipation, confusion, satisfaction)

### 2. Information architecture

Design the product's information structure:

- **Page hierarchy**: Main pages → Sub-pages → Detail pages
- **Navigation structure**: Primary nav, secondary nav, breadcrumbs
- **Content classification**: Information grouping and tagging system
- **Search and filtering**: Information retrieval strategy

### 3. Core flow diagrams

Describe core flow diagrams in text (Mermaid or ASCII):

- User registration/login flow
- Core feature flows (one per major feature)
- Error handling flows

Via `sessions_send`, submit user journeys and information architecture to the CTO for confirmation.
