# Step 3: Execute Changes

**Progress: Step 3 of 4** — Next: Save and Report

## Objective

Apply changes to the identified sections, ensuring document consistency.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps
- Maintain consistent style throughout the PRD when making changes

## Execution Instructions (sequential, no skipping)

### 1. Apply primary changes

Following the impact table from Step 2, modify each corresponding section:

- Add: Insert new content following existing format and style
- Modify: Update in place, preserving unchanged parts
- Delete: Remove content and clean up related references

### 2. Update affected acceptance criteria

If user stories have changed, synchronize the corresponding acceptance criteria:

- New stories: Write Given/When/Then acceptance criteria
- Modified stories: Adjust acceptance criteria
- Deleted stories: Remove corresponding acceptance criteria

### 3. Update affected user stories

If functional requirements have changed, synchronize the corresponding user stories:

- Ensure As a / I want / So that format is correct
- Ensure INVEST principles still hold

### 4. Consistency check

Check the entire PRD for consistency:

- Are requirement numbers sequential?
- Are cross-references correct?
- Is the priority distribution reasonable?
- Does the risk matrix need updating?

## Completion Criteria

- ✅ All identified changes applied
- ✅ Affected acceptance criteria and user stories synchronized
- ✅ Document consistency confirmed

## Next Step
