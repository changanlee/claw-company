# Step 3: Core Requirements

**Progress: Step 3 of 8** — Next: User Stories

## Objective

Extract functional requirements from input documents, structure them into feature groups, and assign priority levels.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions (sequential, no skipping)

### 1. Extract functional requirements

Extract all explicit and implicit functional requirements from the product brief, brainstorming results, and other input documents.

### 2. Structure into feature groups

Organize requirements into logical feature groups, each containing:

- Group name
- Group description
- List of specific features

### 3. Define priority levels (MoSCoW)

Assign priority to each feature:

| Level | Meaning | Description |
|-------|---------|-------------|
| **Must** | Must have | Cannot launch without it |
| **Should** | Should have | Important but can be deferred |
| **Could** | Could have | Nice to have |
| **Won't** | Won't have | Explicitly excluded this time |

### 4. Write to PRD

Write the feature groups and priorities into the PRD "Functional Requirements" section.

### 5. Update PRD frontmatter

Add `step-03-requirements` to `steps-completed`.

## Completion Criteria

- ✅ All functional requirements extracted
- ✅ Requirements structured into logical feature groups
- ✅ Each feature assigned a MoSCoW priority
- ✅ PRD updated
