# Step 4: User Stories

**Progress: Step 4 of 8** — Next: Acceptance Criteria

## Objective

Write INVEST-compliant user stories for each core requirement.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions (sequential, no skipping)

### 1. Write user stories

Write user stories for each Must and Should level requirement, using the format:

```
As a {role},
I want {feature},
So that {value/purpose}.
```

### 2. Ensure INVEST principles

Verify each user story meets the INVEST criteria:

| Principle | Description |
|-----------|-------------|
| **I** — Independent | Stories are as independent as possible |
| **N** — Negotiable | Details are negotiable |
| **V** — Valuable | Clear value to the user |
| **E** — Estimable | Effort can be estimated |
| **S** — Small | Can be completed within one Sprint |
| **T** — Testable | Can be verified as complete |

### 3. Write to PRD

Write the user stories into the PRD "User Stories" section, organized by feature group.

### 4. Update PRD frontmatter

Add `step-04-user-stories` to `steps-completed`.

## Completion Criteria

- ✅ All Must/Should requirements have corresponding user stories
- ✅ Each story meets INVEST principles
- ✅ PRD updated

## Next Step

After confirming completion criteria are met, read and follow: `./step-05-acceptance.md`
