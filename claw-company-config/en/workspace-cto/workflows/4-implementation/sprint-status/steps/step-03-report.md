# Step 3: Produce Status Summary

**Progress: Step 3 of 3** — Final step

## Goal

Produce a concise Sprint status summary including progress, blockers, and risk alerts.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions

### 1. Produce Status Summary

Compile the following:

**Progress Overview:**
- Completion rate percentage (with visual progress bar)
- Story count distribution by status
- Changes from last report (if available)

**Blockers:**
- List all blocked stories
- Blocking reasons
- Suggested resolution approaches

**Risk Alerts:**
- Ordered by risk level
- Impact scope of each risk
- Suggested countermeasures

### 2. Produce Recommendations

Based on analysis results, provide recommendations:

- Stories that should be prioritized next
- Items requiring CTO decision
- Whether Sprint scope adjustment is needed

### 3. Report to CTO

Compile the status summary into a concise report format for CTO.

## Completion Criteria

- ✅ Status summary produced (progress + blockers + risks)
- ✅ Recommendations provided
- ✅ Report delivered to CTO
