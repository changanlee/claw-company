# Step 2: Analyze and Synthesize

**Progress: Step 2 of 3** — Next: Produce retrospective report

## Goal

From the collected data, analyze what went well, what needs improvement, action items, and lessons learned.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions

### 1. What Went Well (Keep)

Identify practices that worked well:

- Which processes ran smoothly
- Which decisions led to good outcomes
- Which tools/methods were particularly effective
- Good habits worth continuing

### 2. What Needs Improvement (Improve)

Identify areas needing improvement:

- Which processes caused delays or issues
- Which problems recurred
- Which communication points were not smooth
- Which estimates were inaccurate

### 3. Action Items

Develop specific actions for each improvement area:

- **What**: Specific improvement action
- **How**: Implementation approach
- **When**: Before next Sprint starts / ongoing improvement
- **Who**: CTO / specific Agent

### 4. Lessons Learned

Extract lessons that can be retained:

- Technical lessons: Architecture / code / testing related
- Process lessons: Planning / execution / review related
- Communication lessons: Collaboration / reporting related
- Determine if each lesson needs routing to other Agents (security → CAO, cost → CFO, process → CHRO)

## Completion Criteria

- ✅ Keep list compiled
- ✅ Improve list compiled
- ✅ Action items developed (with owner and timeline)
- ✅ Lessons extracted (with routing determination)

## Next Step

After confirming completion criteria are met, read and follow: `./step-03-document.md`
