# Step 1: Collect Data

**Progress: Step 1 of 3** — Next: Analyze and synthesize

## Goal

Collect all data from the Sprint or Epic execution process to prepare for analysis.

## Execution Rules

- 📖 Read the entire step file before taking action
- 🚫 Do not pre-read subsequent steps
- 🚫 Do not skip or merge steps

## Execution Instructions

### 1. Collect Completed Stories

Read the sprint-status file and compile:

- All stories' final statuses
- Stories completed as planned
- Stories added outside the plan (if course correction occurred)
- Incomplete stories and reasons

### 2. Collect Issue Records

Collect issues from the following sources:

- Dev Agent Records in story files
- Issues from Code Review reports
- Course correction records (if any)
- Related records in CTO MEMORY.md

### 3. Collect Timeline

Compile a timeline of key events:

- Sprint start and end
- Start and completion times for each story (if recorded)
- Major issue occurrence times
- Course correction times (if any)

### 4. Collect Metrics

Calculate the following metrics:

- Completion rate: Completed stories / planned stories
- Review rejection rate: Rejections / total reviews
- Blocker count
- Scope change count

## Completion Criteria

- ✅ All stories' final statuses compiled
- ✅ Issue records collected
- ✅ Timeline established
- ✅ Metrics calculated

## Next Step

After confirming completion criteria are met, read and follow: `./step-02-analyze.md`
