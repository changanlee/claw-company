# Step 1: Analyze Documentation Status

**Progress: Step 1 of 3** — Next: Write documentation

## Objective

Analyze the project codebase and existing documentation, identify documentation gaps, and create a documentation writing plan.

## Execution Rules

- 📖 Read the entire step before acting
- 🚫 Don't pre-read next steps
- 🚫 Don't skip steps

## Execution Instructions

### 1. Scan Project Structure

Browse the project directory structure to understand:

- Overall project architecture
- Major modules and components
- Tech stack (languages, frameworks, tools)
- Existing documentation files

### 2. Evaluate Existing Documentation

Check existing documentation:

- Does README.md exist and is it complete
- Does API documentation exist
- Does a development guide exist
- Quality of in-code comments

### 3. Identify Documentation Gaps

List missing or incomplete documentation:

| Document Type | Status | Priority |
|--------------|--------|----------|
| README | Missing/Incomplete/Complete | High/Medium/Low |
| API Docs | Missing/Incomplete/Complete | High/Medium/Low |
| Dev Guide | Missing/Incomplete/Complete | High/Medium/Low |
| Architecture | Missing/Incomplete/Complete | High/Medium/Low |
| Deployment Guide | Missing/Incomplete/Complete | High/Medium/Low |

### 4. Create Writing Plan

Prioritize the documentation writing order:

1. README (project entry point)
2. API documentation (developer need)
3. Development guide (onboarding)
4. Other documentation

## Completion Criteria

- ✅ Project structure understood
- ✅ Existing documentation evaluated
- ✅ Documentation gaps identified
- ✅ Writing plan created

## Next Step

After confirming all completion criteria are met, read and follow: `./step-02-write.md`
