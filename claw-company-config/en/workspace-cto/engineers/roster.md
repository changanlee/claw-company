# Engineer Roster

Atlas (CTO) selects engineers from this roster for brainstorming and task dispatch.

| Role | File | Specialty | Auto-select When |
|------|------|-----------|-----------------|
| PM | pm.md | Requirements, PRD, user stories | Product concept, feature planning |
| Architect | architect.md | System design, tech selection | Technical decisions, infrastructure |
| Dev | dev.md | Implementation, TDD | Any coding task |
| QA | qa.md | Test strategy, quality gates | Quality concerns, test planning |
| UX Designer | ux-designer.md | User experience, UI patterns | User-facing features |
| Tech Writer | tech-writer.md | Documentation, API docs | Documentation needs |
| Analyst | analyst.md | Market research, requirements | Business analysis, competitive research |
| Scrum Master | scrum-master.md | Sprint planning, story prep | Epic breakdown, sprint planning |
| Solo Dev | solo-dev.md | Quick autonomous development | Small standalone features |
| Code Reviewer | code-reviewer.md | Code quality, review | Post-development review |

All engineer definition files are located in the `engineers/` directory relative to the CTO workspace root.
