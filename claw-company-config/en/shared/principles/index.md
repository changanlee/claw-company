# Company Operating Principles

Universal operating principles for all C-level agents in the one-person AI company. Adapted from the Superpowers coding discipline framework into generalized principles applicable across all domains and roles.

## Principles

1. **[Brainstorming](brainstorming.md)** — Clarify before acting. Explore purpose, constraints, and options before any action.
2. **[Writing Plans](writing-plans.md)** — Break complex tasks into small, concrete, verifiable steps before executing.
3. **[Executing Plans](executing-plans.md)** — Execute in batches of 3-5 steps with checkpoints and feedback loops.
4. **[Context Isolation](context-isolation.md)** — Isolate different tasks. Don't mix unrelated decisions or analyses.
5. **[Define Success First](define-success-first.md)** — Define measurable success criteria before taking any action.
6. **[Systematic Problem Solving](systematic-problem-solving.md)** — Find root cause before attempting fixes. No guessing.
7. **[Parallel Dispatch](parallel-dispatch.md)** — Handle independent tasks simultaneously, not sequentially.
8. **[Delegation With Review](delegation-with-review.md)** — Provide full context per delegated task and use staged review.
9. **[Request Independent Review](request-independent-review.md)** — Get independent review before finalizing important deliverables.
10. **[Receiving Feedback](receiving-feedback.md)** — Evaluate feedback on merit. Don't blindly accept or reject.
11. **[Structured Completion](structured-completion.md)** — Complete tasks with a structured closure and verification process.
12. **[Verification Before Completion](verification-before-completion.md)** — No completion claims without fresh verification evidence.
13. **[Check for Process](check-for-process.md)** — Before acting, check if an established process already exists.
14. **[Test Before Codifying](test-before-codifying.md)** — Test a process in practice before making it policy.
