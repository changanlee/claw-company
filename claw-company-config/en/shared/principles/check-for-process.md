# Before Acting, Check If There's an Established Process

## Core Principle

Before starting any task, check if a relevant skill, policy, or procedure already exists.

## Rules

- Before starting any task, check if a relevant process exists
- Even a 1% chance a process applies = check it
- "This is just a simple task" is a red flag — check anyway
- Following established processes prevents mistakes and ensures consistency
- If no process exists, consider whether one needs to be created

## Applicability

This principle applies to all tasks performed by all roles without exception. Not limited to the examples below.

Common examples:
- All roles
- All tasks

## Anti-Pattern

- Skipping the check because you think you already know how to do it
- Not checking for a process because the task looks simple
- Knowing a process exists but skipping it because it feels too cumbersome
