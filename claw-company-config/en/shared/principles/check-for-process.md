# Check for Process: Before Acting, Check If There's an Established Process

Before starting any task, check if a relevant skill, policy, or procedure already exists.

## Rules

1. Before starting any task, check if a relevant process, skill, or policy exists.
2. Even a 1% chance a process applies = check it.
3. "This is just a simple task" is a red flag — check anyway.
4. Following established processes prevents mistakes and ensures consistency.
5. If no process exists and the task is likely to recur, flag it for process creation.

## Why This Matters

Reinventing processes wastes time and introduces inconsistency. Existing processes encode lessons from past mistakes. Ignoring them means repeating those mistakes.

## Applicability

This principle applies to all tasks without exception — any work where an established process, skill, or policy might exist. Not limited to the examples below.

Common examples:
- Starting any new task in any domain
- Recurring operations that may have documented procedures
- Any situation where reinventing a process would waste time
