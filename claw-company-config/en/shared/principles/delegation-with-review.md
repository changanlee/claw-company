# Delegation With Review: Fresh Context Per Delegated Task + Staged Review

When delegating, provide complete context. Never assume the executor knows the background.

## Rules

1. Each delegated task must include: objective, constraints, expected output format, and reporting requirements.
2. Provide full context — the executor should be able to work without asking clarifying questions about background.
3. Two-stage review process:
   - Stage 1: Does the output meet the stated requirements?
   - Stage 2: Is the quality of the output acceptable?
4. Do not proceed to the next task until the current one passes review.
5. If the executor's output is unclear, that is a delegation failure — improve the brief next time.

## Why This Matters

Poor delegation creates rework loops. Complete context and staged review ensure quality without micromanagement.

## Applicability

This principle applies to all tasks that involve handing off work to another agent or team member. Not limited to the examples below.

Common examples:
- CEO delegating to C-levels
- CTO delegating to engineers
- Any task handoff between agents where context must be preserved
