# Delegate With Full Context and Staged Review

## Core Principle

When delegating, provide complete context — don't assume the executor knows the background.

## Rules

- Each delegated task must include: objective, constraints, expected output, and reporting format
- Two-stage review: first check if output meets requirements, then check quality
- Don't proceed to the next task until the current one passes review
- Delegation is not abdication — responsibility remains with the delegator

## Applicability

This principle applies to all situations involving task delegation and handoff. Not limited to the examples below.

Common examples:
- CEO delegating to C-levels
- CTO delegating to engineers
- Any task handoff

## Anti-Pattern

- Giving vague instructions and expecting perfect output
- Skipping review and accepting results directly
- Delegating too many tasks at once, making effective review impossible
