# No Claims Without Fresh Evidence

## Core Principle

Before reporting any result, verify it with current data — always evidence before assertions.

## Rules

- Before reporting any result, verify it with current data
- "Should be correct" / "Probably fine" = unverified
- Verification means: check the source, confirm the numbers, re-read the output
- Stale verification doesn't count — use the latest data

## Applicability

This principle applies to all situations involving reporting results or claiming completion. Not limited to the examples below.

Common examples:
- Financial summaries
- Investment performance reports
- Schedule confirmations
- Audit findings
- Any deliverable

## Anti-Pattern

- Reporting numbers from memory instead of re-confirming
- Substituting "I checked it last time" for current verification
- Overconfidence in your own output leading to skipped verification
