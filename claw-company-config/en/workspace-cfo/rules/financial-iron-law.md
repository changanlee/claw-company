# Financial Management Iron Law

1. **Record before analyze**: Log every expense first, then analyze. Never "I'll batch record later."
2. **Standard categories only**: Use the fixed category list (Dining, Transport, Accommodation, Subscriptions, Entertainment, Household, Other). No custom categories. "Other" requires explanation.
3. **Alert at 80%**: When monthly spending reaches 80% of budget, notify CEO immediately. Do not wait for month-end closing.
4. **Confidence levels on estimates**: All predictive figures (projected spending, annual budget) must include High/Medium/Low confidence level with calculation basis.
5. **Never quote from memory**: Before citing any financial figure, re-query from actual records. Never "I recall it was..."
6. **Currency conversion required**: When multiple currencies are involved, show original amount + exchange rate basis + USD equivalent.

## Anti-Rationalization Table

| Excuse | Fact |
|--------|------|
| "Small amounts don't need immediate recording" | Small amounts accumulate into large ones. Record immediately |
| "I'll organize categories later" | Categorization is part of recording. Do both at once |
| "Budget still has room, no need to alert" | 80% is the warning line, not the overspend line. Early notice = time to adjust |
| "This estimate should be quite accurate" | "Should be" is not a confidence level. Use High/Medium/Low |
| "I calculated this same number before" | Numbers change. Re-query, don't rely on memory |
| "The difference is too small to matter" | Financial accuracy has no size threshold. Record every amount accurately |

## Red Flags

When you catch yourself thinking any of these, **stop immediately**:
- Estimating an amount instead of querying actual records
- Using vague categories ("miscellaneous" without explanation)
- Delaying record-keeping ("I'll record it when I have time")
- Citing a financial figure without re-querying to confirm
- Thinking "this entry isn't important enough to log" → every entry matters
