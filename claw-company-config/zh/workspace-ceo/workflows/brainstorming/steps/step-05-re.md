# 步驟 5：PM3 — 逆向工程

**進度：步驟 5 / 共 7 步**

## 📌 Phase 2 — Reverse Engineering（逆向工程）

## 目標

拆解現有實作（或目標狀態），理解每個元件的存在理由與交互方式。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步
- 🔒 董事長確認後才進入下一步

## 執行指令

### 1. 拆解現狀

引導問題（逐一與董事長討論）：

- 「現在是**怎麼運作**的？」
- 「每個部分**為什麼**要這樣設計？」
- 「有哪些**隱含的依賴**或副作用？」
- 「哪些部分如果移除，整體會**崩潰**？」

### 2. 按需 Spawn 專家

遇到技術細節或領域知識時，用 `sessions_spawn` + 脈絡快照拉入專家。

此階段特別適合 spawn：
- **CTO / Architect** — 技術可行性、系統依賴分析
- **CFO** — 成本結構拆解
- **Analyst** — 市場現狀研究

### 3. 產出拆解圖

整理為結構化分析：

```
## 現狀拆解

| 元件 | 存在理由 | 依賴 | 風險 |
|------|---------|------|------|
| [元件 1] | [為什麼存在] | [依賴什麼] | [可能的問題] |
| [元件 2] | ... | ... | ... |

### 關鍵發現
- [發現 1]
- [發現 2]
```

### 4. 確認推進

向董事長呈現分析，確認：
> Phase 2 拆解完成。現狀分析如上，是否滿意？確認後進入 Phase 3 批判精煉。

## 完成標準

- [ ] 已拆解現狀或目標狀態
- [ ] 每個元件的存在理由和依賴已明確
- [ ] 董事長確認推進
