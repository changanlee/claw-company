# 步驟 3：發散探索

**進度：步驟 3 / 共 7 步**

## 目標

執行選定的思考技法，盡量拓展廣度。過程中按需 spawn 專家提供領域觀點。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步
- ⚡ 發散階段不評判想法好壞，只求數量和多元性

## 執行指令

### 1. 執行技法

對每個選定的技法（模式 4 跳過此步）：

1. 向董事長說明技法規則和引導問題（從 CSV 的 description 欄位取得）。
2. 引導發散思考，鼓勵大膽想法。
3. 每個技法執行 **1-2 輪**對話。
4. **反偏見機制**：每累積 10 個想法時，刻意切換思考角度。

### 2. 按需 Spawn 專家

討論過程中，遇到以下情境時 spawn 專家進行定點深潛：

**自動觸發（CEO 判斷）：**

| 觸發情境 | Spawn 對象 | Agent ID |
|---------|-----------|----------|
| 涉及成本或預算估算 | CFO | cc-cfo |
| 涉及市場趨勢或投資 | CIO | cc-cio |
| 涉及行程或生活影響 | COO | cc-coo |
| 涉及系統架構或技術可行性 | CTO 或 Architect | cc-cto |
| 涉及組織設計或人力 | CHRO | cc-chro |
| 涉及合規或風險 | CAO | cc-cao |

**手動觸發：**
- 董事長隨時可以說「拉 XX 進來」「問一下 CTO」

**Spawn 脈絡快照格式：**

使用 `sessions_spawn` 時，task prompt 必須包含：

```
## 腦力激盪脈絡

### 議題
[議題定義]

### 目前進度
[已產生的關鍵想法摘要]

### 需要你回答的問題
[具體問題 1-3 個]

### 回覆格式
請直接回答上述問題，附上你的專業分析。簡潔為主，不超過 500 字。
```

### 3. 整合專家觀點

收到 spawn 專家的回覆後：
1. 將觀點整合進討論。
2. 向董事長呈現：「[角色] 的觀點是 [摘要]，這帶出了 [新的思考方向]」。
3. 繼續發散。

### 4. 記錄想法

記錄所有產出的想法，按主題分組。這份清單將作為 PM3 收斂階段的輸入。
