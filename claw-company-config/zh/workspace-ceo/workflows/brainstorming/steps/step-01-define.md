# 步驟 1：議題定義

**進度：步驟 1 / 共 5 步**

## 目標

清晰定義腦力激盪的議題範圍，並邀請相關高管加入討論。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 參考原則

讀取 `{{INSTALL_DIR}}/shared/principles/brainstorming.md` — 行動前先釐清問題，探索多種可能性再收斂。

## 執行指令

### 1. 議題定義

與董事長確認：

- 議題核心問題
- 期望產出（決策方向 / 創意方案 / 問題解法）
- 範圍邊界（哪些不在討論範圍內）

### 2. 邀請高管

根據議題性質，使用 `sessions_send` 邀請相關高管：

| 議題類型 | 建議邀請 |
|---------|---------|
| 財務/成本相關 | CFO |
| 投資/市場相關 | CIO |
| 生活/營運相關 | COO |
| 技術/產品相關 | CTO |
| 組織/人力相關 | CHRO |
| 風險/合規相關 | CAO |
| 戰略性議題 | 全員 |

### 3. 向高管說明議題

發送議題背景和期望貢獻方向給受邀高管。

## 完成標準

- [ ] 議題已清晰定義
- [ ] 已邀請相關高管
- [ ] 高管已收到議題說明
