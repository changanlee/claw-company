# 步驟 4：PM3 — 第一性原理

**進度：步驟 4 / 共 7 步**

## 📌 Phase 1 — First Principles（第一性原理）

## 目標

從發散階段產生的想法清單中，拆解到最基本的事實與約束，消除假設。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步
- 🔒 董事長確認後才進入下一步

## 執行指令

### 1. 回顧發散結果

向董事長呈現步驟 3 的想法清單摘要，確認這些是收斂的輸入。

### 2. 拆解基本事實

引導問題（逐一與董事長討論）：

- 「這個議題的**本質**是什麼？」
- 「哪些是**事實**、哪些是**假設**？」
- 「如果從零開始設計，什麼是**不可省略**的？」
- 「哪些限制是**真正的約束**，哪些只是慣例？」

### 3. 按需 Spawn 專家

同步驟 3 的 spawn 機制 — 遇到需要領域深度的問題時，用 `sessions_spawn` + 脈絡快照拉入專家。

### 4. 產出事實清單

整理為結構化清單：

```
## 基本事實與核心約束

### 事實（已驗證）
- F1: [事實]
- F2: [事實]

### 約束（硬限制）
- C1: [約束]
- C2: [約束]

### 假設（待驗證）
- A1: [假設] — 驗證方式：[如何驗證]
```

### 5. 確認推進

向董事長呈現清單，確認：
> Phase 1 拆解完成。事實清單如上，是否滿意？確認後進入 Phase 2 逆向工程。

## 完成標準

- [ ] 已拆解出基本事實
- [ ] 已區分事實、約束、假設
- [ ] 董事長確認推進
