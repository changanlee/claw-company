# 步驟 1：分析指令

**進度：步驟 1 / 共 3 步**

## 目標

解析董事長指令的類型、急迫度和涉及部門，為後續路由提供依據。

- 📖 讀完整個步驟檔案再開始行動

## 參考原則

讀取 `{{INSTALL_DIR}}/shared/principles/delegation-with-review.md` — 分派任務時附完整脈絡，設定明確期望產出與回報時限。

## 執行指令

### 1. 指令類型分類

將董事長的指令歸類至以下類型之一：

| 類型 | 說明 | 對應高管 |
|------|------|---------|
| 財務 | 記帳、預算、消費、成本 | CFO |
| 投資 | 投資組合、市場分析、買賣決策 | CIO |
| 生活 | 行程、飲食、出行、天氣 | COO |
| 技術 | 產品開發、程式碼、系統設計 | CTO |
| Agent 管理 | 能力評估、政策、模型調整 | CHRO |
| 安全/稽核 | 安全掃描、合規檢查 | CAO |
| 綜合 | 涉及多部門 | CEO 拆分後分派 |

### 2. 急迫度判斷

- **緊急**：包含「馬上」「立刻」「ASAP」等關鍵字，或董事長明確要求
- **一般**：無特殊時間壓力
- **低優先**：資訊收集、長期規劃

### 3. 涉及部門識別

- 單一部門：直接路由
- 多部門：拆分子任務，分別路由
