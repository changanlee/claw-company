# CEO 高管策略腦力激盪

## 觸發條件

- **主動觸發**：董事長明確要求（「我想 brainstorm 一下 XXX」「幫我想想 XXX 的方向」）
- **建議觸發**：CEO 收到模糊、策略性、多部門交叉的需求時，主動向董事長建議：
  > 「這個議題涉及 [原因]，建議先做一次腦力激盪再決定方向，您覺得如何？」

董事長同意後才啟動流程。

---

## 第一步：議題定義

1. 確認腦力激盪的**主題**和**目標**。
2. 判斷議題類型：
   - **決策型**：需要在幾個方向中選一個（產出：決策摘要 + 行動項目）
   - **探索型**：開放式探索可能性（產出：創意清單 + 洞察）
3. 根據議題，用 `sessions_send` 邀請相關高管加入討論：
   - 技術方向 → CTO
   - 投資/商業 → CIO
   - 財務影響 → CFO
   - 營運影響 → COO
   - 組織/人才 → CHRO
   - 至少拉入 **2 個**相關高管。

---

## 第二步：選擇技法模式

讀取 `{{INSTALL_DIR}}/shared/brain-methods.csv`，向董事長提供四種模式選擇：

> 接下來的腦力激盪，您想用哪種方式選擇思考技法？
>
> 1. **自選** — 瀏覽 9 大分類，挑您感興趣的技法
> 2. **AI 推薦** — 我根據議題推薦 2-3 個最適合的技法
> 3. **隨機** — 隨機抽取，用驚喜激發靈感
> 4. **漸進式** — 從發散到收斂，自動串接多個技法

---

## 第三步：技法選擇

### 模式 1 — 自選

1. 列出 9 大分類及各分類的技法數量：
   - collaborative（5）、creative（11）、deep（8）、introspective_delight（6）、structured（7）、theatrical（6）、wild（8）、biomimetic（3）、quantum（3）、cultural（4）
2. 董事長選分類後，列出該分類所有技法的名稱和簡述。
3. 董事長挑選 1-3 個技法。

### 模式 2 — AI 推薦

根據議題分析以下維度，推薦 2-3 個技法：
- **目標分析**：創新探索 → creative/wild 類；問題解決 → deep/structured 類
- **複雜度**：複雜抽象 → deep/structured；具體明確 → creative/theatrical
- **時間**：短時間（< 30 分鐘）→ 1 個技法；長時間 → 多技法串接
- 附上推薦理由，董事長可接受或換選。

### 模式 3 — 隨機

從 CSV 中隨機抽取 2 個技法，呈現給董事長：
- 展示名稱、分類、說明
- 董事長可接受或要求重抽

### 模式 4 — 漸進式

自動設計多階段序列：
1. **基礎發散** — 從 creative 或 collaborative 類選 1 個技法
2. **深入挖掘** — 從 deep 或 structured 類選 1 個技法
3. **整理收斂** — 從 structured 類選 1 個技法
每階段完成後自動進入下一階段。

---

## 第四步：執行技法

對每個選定的技法：

1. 向參與者說明技法的規則和引導問題（從 CSV 的 description 欄位取得）。
2. CEO 主持討論：
   - 先請各高管從各自專業角度發表觀點（用 `sessions_send` 收集）。
   - 整合各高管觀點後呈現給董事長。
   - 引導董事長參與回應和延伸。
3. 每個技法執行 **1-2 輪**對話，記錄所有產出的想法。
4. **反偏見機制**：每累積 10 個想法時，刻意切換思考領域，避免語義聚集。

---

## 第五步：整理產出

根據第一步判斷的議題類型：

### 決策型產出

```
## 腦力激盪摘要

### 議題
[主題]

### 參與者
CEO + [高管列表]

### 使用技法
[技法列表]

### 關鍵洞察
- [洞察 1]
- [洞察 2]

### 決策選項
| 選項 | 優點 | 缺點 | 推薦程度 |
|------|------|------|---------|
| A    |      |      |         |
| B    |      |      |         |

### 建議行動項目
1. [行動 1] — 負責人：[角色]
2. [行動 2] — 負責人：[角色]
```

### 探索型產出

```
## 腦力激盪摘要

### 議題
[主題]

### 參與者
CEO + [高管列表]

### 使用技法
[技法列表]

### 創意清單
按主題分群：
#### 群組 A：[主題]
- [想法 1]
- [想法 2]

#### 群組 B：[主題]
- [想法 3]

### 發現的模式與洞察
- [洞察 1]
- [洞察 2]

### 值得深入探索的方向
- [方向 1]
- [方向 2]
```

---

## 第六步：呈報與後續

1. 將整理後的摘要呈報董事長。
2. 根據董事長回饋決定後續：
   - 需要執行 → 按分派原則指派給對應高管
   - 需要更深入 → 可再次執行本流程或轉入專項研究
   - 存檔備查 → 記錄到 CEO memory
