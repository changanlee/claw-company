# 工兵角色清單

Atlas（CTO）可依據任務需求召喚以下工兵角色。每個角色有獨立的定義檔案，描述其身份、能力範圍與工作方式。

## 可用角色

| 角色 | 檔案 | 專長 | 自動加入時機 |
|------|------|------|-------------|
| PM | pm.md | 需求分析、PRD、用戶故事 | 產品構想、功能規劃 |
| 架構師 | architect.md | 系統設計、技術選型 | 技術決策、基礎設施 |
| 開發工兵 | dev.md | 實作、TDD | 任何編碼任務 |
| QA 工兵 | qa.md | 測試策略、品質閘門 | 品質問題、測試規劃 |
| UX 設計師 | ux-designer.md | 使用者體驗、UI 模式 | 面向用戶的功能 |
| 技術文件工兵 | tech-writer.md | 文件撰寫、API 文件 | 文件需求 |
| 分析師 | analyst.md | 市場研究、需求分析 | 商業分析、競爭研究 |
| Scrum Master | scrum-master.md | Sprint 規劃、故事準備 | Epic 拆解、Sprint 規劃 |
| 獨立開發工兵 | solo-dev.md | 快速自主開發 | 小型獨立功能 |
| Code Reviewer | code-reviewer.md | 代碼品質、審查 | 開發後審查 |

## 使用方式

- Atlas 根據任務性質自動選擇合適的工兵角色
- 董事長可隨時要求追加角色（例如：「讓 QA 也來看看」）
- 每個工兵 spawn 時會自動注入適用的鐵律規則
- 所有工兵完成任務後必須依照統一回報格式回報結果
