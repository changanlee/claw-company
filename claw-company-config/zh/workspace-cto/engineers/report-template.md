# 工程師回報模板

所有工程師（Sub-Agent）完成任務後，必須依照以下格式回報結果給 CTO。

## 回報格式

```
## 任務結果

- 狀態：成功 / 部分成功 / 失敗
- 摘要：<一句話描述完成了什麼>
- 產出物：<PR 連結 / 檔案清單 / 文件連結>

## 遇到的問題

- <問題 1 描述 + 如何解決>
- <問題 2 描述 + 如何解決>
- （無問題則寫「無」）

## 教訓與建議

- <值得記錄的技術發現、模式、或踩坑經驗>
- <對未來類似任務的建議>
- （無則寫「無」）

## 測試驗證

- 測試狀態：通過 / 部分通過 / 未測試
- 測試覆蓋：<描述測試了什麼>
- 未覆蓋項目：<描述未測試的部分及原因>

## 資源消耗

- 預估 Token 消耗：<低/中/高>
- 是否需要後續工作：是 / 否
- 後續工作描述：<如有>
```

## CTO 收到回報後的處理流程

1. 審核成果品質
2. 提取教訓 → 寫入 CTO 的 MEMORY.md（如為長期有效的模式）
3. 更新 status.md 任務狀態
4. 記錄到 memory/YYYY-MM-DD.md 日誌
5. 精煉摘要 → 向 CEO 回報（提及同事時遵照 `{{INSTALL_DIR}}/shared/team-roster.md` 稱呼規則）
6. 如涉及安全問題 → 通知 CAO
