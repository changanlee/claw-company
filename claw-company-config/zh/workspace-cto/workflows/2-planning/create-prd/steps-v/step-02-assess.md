# 步驟 2：品質評估

**進度：步驟 2 / 共 3 步** — 下一步：產出驗證報告

## 目標

對每個區段進行品質評級，識別缺口與不一致之處。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 區段品質評級

對每個區段標記評級：

| 區段 | 評級 | 說明 |
|------|------|------|
| 專案概述 | ✅ Pass / ⚠️ Needs improvement / ❌ Missing | {具體說明} |
| 功能需求 | ✅ / ⚠️ / ❌ | {具體說明} |
| 用戶故事 | ✅ / ⚠️ / ❌ | {具體說明} |
| 驗收標準 | ✅ / ⚠️ / ❌ | {具體說明} |
| 非功能性需求 | ✅ / ⚠️ / ❌ | {具體說明} |
| 邊界條件與風險 | ✅ / ⚠️ / ❌ | {具體說明} |

### 2. 識別缺口

列出所有未完成或不足的項目：

- **缺失（❌）**：完全沒有涵蓋的區段或項目
- **不足（⚠️）**：有涵蓋但品質不達標的項目，說明具體問題

### 3. 檢查一致性

跨區段一致性檢查：

- 功能需求 ↔ 用戶故事 ↔ 驗收標準是否一一對應
- NFR 是否與功能需求的規模匹配
- 風險是否涵蓋了關鍵技術決策

## 完成條件

- ✅ 每個區段已評級
- ✅ 缺口已識別並記錄
- ✅ 一致性已檢查

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-03-report.md`
