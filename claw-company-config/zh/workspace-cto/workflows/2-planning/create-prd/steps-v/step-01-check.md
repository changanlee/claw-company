# 步驟 1：載入與逐項驗證

**進度：步驟 1 / 共 3 步** — 下一步：品質評估

## 目標

載入 PRD 和驗證清單，逐項檢查每個區段的完整性和品質。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 載入 PRD

讀取 `{{INSTALL_DIR}}/workspace-cto/output/planning/prd.md`。

如果 CTO 指定了其他路徑，使用指定路徑。

### 2. 逐項驗證

按以下清單檢查每個項目：

**專案概述**
- [ ] 專案類型已分類
- [ ] 複雜度已評估
- [ ] Greenfield / Brownfield 已判斷

**功能需求**
- [ ] 需求已結構化為功能群組
- [ ] 每個需求有 MoSCoW 優先級
- [ ] Must 級別需求明確且合理

**用戶故事**
- [ ] 每個 Must/Should 需求有對應的用戶故事
- [ ] 故事使用 As a / I want / So that 格式
- [ ] 故事符合 INVEST 原則

**驗收標準**
- [ ] 每個用戶故事有驗收標準
- [ ] 使用 Given/When/Then 格式
- [ ] 包含正常路徑和邊界條件

**非功能性需求**
- [ ] 效能需求有可量測指標
- [ ] 安全需求已定義
- [ ] 可用性目標已設定

**邊界條件與風險**
- [ ] 邊界條件已識別
- [ ] 風險矩陣已建立
- [ ] 中/高風險有緩解策略

### 3. 額外品質檢查

- [ ] 需求是否可測試？（能否從需求直接導出測試案例）
- [ ] 驗收標準是否足夠具體？（無模糊詞彙如「快速」、「友善」）
- [ ] NFR 是否可量測？（有明確數字，非「高效能」之類的形容）

## 完成條件

- ✅ PRD 已完整載入
- ✅ 所有驗證項目已逐一檢查

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-02-assess.md`
