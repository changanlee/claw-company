# 步驟 3：核心需求

**進度：步驟 3 / 共 8 步** — 下一步：用戶故事

## 目標

從輸入文件中提取功能需求，結構化為功能群組並排列優先級。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 提取功能需求

從產品簡介、腦力激盪結果等輸入文件中，提取所有明確和隱含的功能需求。

### 2. 結構化為功能群組

將需求組織為邏輯功能群組，每個群組：

- 群組名稱
- 群組描述
- 包含的具體功能列表

### 3. 定義優先級（MoSCoW）

對每個功能標記優先級：

| 級別 | 含義 | 說明 |
|------|------|------|
| **Must** | 必須 | 沒有就不能上線 |
| **Should** | 應該 | 重要但可延後 |
| **Could** | 可以 | 錦上添花 |
| **Won't** | 不做 | 本次明確不做 |

### 4. 寫入 PRD

將功能群組和優先級寫入 PRD 的「功能需求」區段。

### 5. 更新 PRD frontmatter

將 `step-03-requirements` 加入 `steps-completed`。

## 完成條件

- ✅ 所有功能需求已提取
- ✅ 需求已分組為邏輯功能群組
- ✅ 每個功能已標記 MoSCoW 優先級
- ✅ PRD 已更新

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-04-user-stories.md`
