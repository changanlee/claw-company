# 步驟 5：驗收標準

**進度：步驟 5 / 共 8 步** — 下一步：非功能性需求

## 目標

為每個用戶故事定義明確的驗收標準，包含邊界條件。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 定義驗收標準

為每個用戶故事撰寫驗收標準，使用 Given/When/Then 格式：

```
Given {前置條件},
When {觸發動作},
Then {預期結果}.
```

### 2. 納入邊界條件

每個驗收標準至少考慮：

- **正常路徑**：標準操作流程
- **邊界值**：極端輸入、空值、最大最小值
- **錯誤路徑**：無效輸入、權限不足、網路失敗

### 3. 寫入 PRD

將驗收標準寫入 PRD，附在對應的用戶故事下方。

### 4. 更新 PRD frontmatter

將 `step-05-acceptance` 加入 `steps-completed`。

## 完成條件

- ✅ 每個用戶故事都有至少一組驗收標準
- ✅ 驗收標準使用 Given/When/Then 格式
- ✅ 邊界條件已考慮
- ✅ PRD 已更新

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-06-nfr.md`
