# 步驟 4：用戶故事

**進度：步驟 4 / 共 8 步** — 下一步：驗收標準

## 目標

為每個核心需求撰寫符合 INVEST 原則的用戶故事。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 撰寫用戶故事

為每個 Must 和 Should 級別的需求撰寫用戶故事，格式：

```
As a {角色},
I want {功能},
So that {價值/目的}.
```

### 2. 確保 INVEST 原則

檢查每個用戶故事是否符合：

| 原則 | 說明 |
|------|------|
| **I** — Independent | 故事之間盡量獨立 |
| **N** — Negotiable | 細節可協商 |
| **V** — Valuable | 對使用者有明確價值 |
| **E** — Estimable | 可估算工作量 |
| **S** — Small | 規模適中，可在一個 Sprint 內完成 |
| **T** — Testable | 可以驗證是否完成 |

### 3. 寫入 PRD

將用戶故事寫入 PRD 的「用戶故事」區段，按功能群組組織。

### 4. 更新 PRD frontmatter

將 `step-04-user-stories` 加入 `steps-completed`。

## 完成條件

- ✅ 所有 Must/Should 需求都有對應的用戶故事
- ✅ 每個故事符合 INVEST 原則
- ✅ PRD 已更新
