# 步驟 6：非功能性需求

**進度：步驟 6 / 共 8 步** — 下一步：邊界條件與風險

## 目標

定義系統的非功能性需求（NFR），確保品質屬性有明確的可量測指標。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 效能需求

定義關鍵效能指標：

- 回應時間（P50 / P95 / P99）
- 吞吐量（RPS / TPS）
- 啟動時間
- 資源使用限制（記憶體、CPU）

### 2. 安全需求

- 認證與授權機制
- 資料加密（傳輸中 / 靜止中）
- 輸入驗證與防禦
- 稽核日誌需求

### 3. 可擴展性需求

- 預期使用者規模
- 資料成長預估
- 水平 / 垂直擴展策略

### 4. 可用性需求

- SLA 目標（99.9% / 99.99%）
- 容錯與故障恢復
- 備份策略

### 5. 合規需求（如適用）

- 資料保護法規（GDPR、個資法）
- 產業特定法規
- 資料留存政策

### 6. 寫入 PRD

將非功能性需求寫入 PRD 的「非功能性需求」區段，每項需求必須可量測。

### 7. 更新 PRD frontmatter

將 `step-06-nfr` 加入 `steps-completed`。

## 完成條件

- ✅ 效能、安全、可擴展性、可用性需求已定義
- ✅ 每項 NFR 有可量測指標
- ✅ 合規需求已評估（如適用）
- ✅ PRD 已更新
