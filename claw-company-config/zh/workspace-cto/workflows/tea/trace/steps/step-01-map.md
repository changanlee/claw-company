# 步驟 1：提取需求與測試

**進度：步驟 1 / 共 3 步** — 下一步：建立對應矩陣

## 目標

從 PRD 提取所有需求 ID，從測試代碼提取所有測試案例。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 提取需求 ID

從 PRD 和相關文件中提取所有需求：

- **功能需求**：帶有 ID 的功能描述（如 FR-001, REQ-001）
- **用戶故事**：帶有 ID 的 user story（如 US-001）
- **驗收標準**：每個故事的 AC 編號
- 如文件無 ID，自行編號建立索引

### 2. 提取測試案例

從測試代碼中提取所有測試案例：

- 掃描所有測試檔案
- 提取測試名稱（describe/it 組合）
- 記錄測試檔案路徑
- 識別測試類型（單元/整合/E2E）

### 3. 整理清單

產出兩份結構化清單：

**需求清單：**
| ID | 描述 | 模組 | 優先級 |
|----|------|------|--------|

**測試清單：**
| 測試名稱 | 檔案路徑 | 類型 | 相關模組 |
|----------|---------|------|---------|

## 完成條件

- ✅ 所有需求 ID 已提取並列表
- ✅ 所有測試案例已提取並列表
- ✅ 兩份清單格式統一、完整

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-02-link.md`
