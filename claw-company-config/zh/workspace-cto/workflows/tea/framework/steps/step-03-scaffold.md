# 步驟 3：建立基礎測試範例

**進度：步驟 3 / 共 3 步** — 最終步驟

## 目標

建立基礎測試腳本和範例，作為團隊開發測試的參考模板。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 建立範例單元測試

建立一個示範性的單元測試檔案，展示：

- AAA 模式（Arrange-Act-Assert）
- 正面和負面測試案例
- 描述性的測試命名
- 適當的 describe/it 分組

### 2. 建立範例整合測試

建立一個示範性的整合測試檔案，展示：

- API 端點測試（如適用）
- 資料庫操作測試（如適用）
- Mock 外部依賴的方式
- setup/teardown 的使用

### 3. 建立測試工具函數

建立共用的測試 helpers：

- 測試資料工廠函數（factories）
- 常用的 mock 函數
- 自定義 matcher / assertion
- 測試環境初始化工具

### 4. 執行驗證

運行所有範例測試，確認：

- 所有範例測試通過
- 覆蓋率報告正常產生
- 無 lint 或型別錯誤

### 5. 交付回報

向 CTO 回報：

- 測試框架已初始化完成
- 選用的框架和版本
- 目錄結構說明
- 範例測試檔案路徑
- 可用的 npm scripts 清單

## 完成條件

- ✅ 範例單元測試已建立並通過
- ✅ 範例整合測試已建立並通過
- ✅ 測試工具函數已建立
- ✅ 已向 CTO 回報結果
