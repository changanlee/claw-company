# 步驟 2：設計測試架構

**進度：步驟 2 / 共 3 步** — 下一步：產出測試計畫文件

## 目標

根據分析結果，設計測試架構，包含測試金字塔比例、框架選擇和覆蓋策略。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 設計測試金字塔

根據專案特性決定測試比例：

- **單元測試**：核心邏輯、工具函數、資料轉換（佔比最高）
- **整合測試**：API 端點、資料庫操作、服務間通訊
- **E2E 測試**：關鍵用戶旅程、付款流程、註冊登入（佔比最低）
- 標註每層的具體測試數量目標

### 2. 選擇測試框架

根據專案技術棧推薦測試框架：

- **單元/整合**：Jest / Vitest / Mocha / pytest / JUnit 等
- **E2E**：Playwright / Cypress / Selenium 等
- **API**：Supertest / REST Assured / httpx 等
- 說明選擇理由（社群支援、學習曲線、專案契合度）

### 3. 制定覆蓋策略

- **覆蓋率目標**：整體目標和各模組目標
- **優先覆蓋區域**：高風險區域優先（對照步驟 1 的風險評估）
- **排除項目**：不需測試的範圍（第三方套件、自動生成代碼）
- **覆蓋類型**：行覆蓋、分支覆蓋、函數覆蓋

### 4. 定義測試環境

- 測試資料策略（fixtures / factories / seeds）
- Mock 策略（哪些依賴需要 mock）
- 測試環境配置（環境變數、資料庫、外部服務）

## 完成條件

- ✅ 測試金字塔比例已確定
- ✅ 測試框架已選定（含理由）
- ✅ 覆蓋策略已制定（目標、優先級、排除項）
- ✅ 測試環境需求已定義
