# 步驟 2：評估 NFR 測試策略

**進度：步驟 2 / 共 3 步** — 下一步：產出 NFR 測試計畫

## 目標

為每個 NFR 評估合適的測試策略和所需工具。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 效能測試策略

對每個效能相關 NFR：

- **負載測試**：模擬正常負載，驗證回應時間
- **壓力測試**：模擬超載情境，驗證系統行為
- **容量測試**：確認系統容量上限
- **工具選擇**：k6 / Artillery / Locust / JMeter
- **環境需求**：測試環境規格、測試資料量

### 2. 安全測試策略

對每個安全相關 NFR：

- **靜態分析**：SAST 工具掃描代碼漏洞
- **動態分析**：DAST 工具掃描運行中的應用
- **依賴掃描**：檢查第三方套件漏洞
- **滲透測試**：模擬攻擊情境
- **工具選擇**：Snyk / OWASP ZAP / SonarQube / npm audit

### 3. 可靠性測試策略

對每個可靠性相關 NFR：

- **故障注入**：模擬服務中斷、網路異常
- **恢復測試**：驗證故障恢復流程
- **備份驗證**：驗證備份和還原程序
- **工具選擇**：Chaos Engineering 工具（如適用）

### 4. 可擴展性測試策略

對每個可擴展性相關 NFR：

- **水平擴展驗證**：增加節點後的效能表現
- **資料增長模擬**：大量資料下的效能表現
- **工具選擇**：與效能測試工具相同 + 容器編排工具

### 5. 評估可行性

對每個 NFR 測試策略評估：

- **自動化可行性**：能否納入 CI/CD
- **成本**：工具授權、環境資源
- **依賴**：需要的外部資源或配合

## 完成條件

- ✅ 每個 NFR 都有對應的測試策略
- ✅ 工具需求已識別
- ✅ 可行性已評估
