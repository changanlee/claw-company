# 步驟 2：實作自動化測試

**進度：步驟 2 / 共 3 步** — 下一步：產出自動化摘要

## 目標

按風險優先級順序，為識別的缺口編寫自動化測試腳本。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 從最高優先級開始

按 P0 → P1 → P2 順序，逐一實作自動化測試。

### 2. 編寫測試代碼

對每個缺口：

- 遵循 AAA 模式（Arrange-Act-Assert）
- 使用專案的測試框架和慣例
- 測試命名清晰描述行為
- 適當使用 mock/stub（不過度 mock）

### 3. 確保測試品質

每個測試必須：

- 獨立可運行（不依賴其他測試的執行順序）
- 包含正面和負面測試案例
- 覆蓋關鍵邊界條件
- 執行速度合理（單元 < 100ms，整合 < 1s）

### 4. 逐一驗證

每完成一個測試：

- 執行測試確認通過（GREEN）
- 確認不影響現有測試
- 更新覆蓋率數據

### 5. 記錄進度

記錄每個已完成的自動化測試：

- 測試檔案路徑
- 覆蓋的功能點
- 測試類型（單元/整合/E2E）

## 完成條件

- ✅ P0 優先級的缺口已全部補齊
- ✅ P1 優先級的缺口已盡可能補齊
- ✅ 所有新增測試通過
- ✅ 現有測試不受影響

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-03-summary.md`
