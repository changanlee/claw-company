# 步驟 2：驗證完整性

**進度：步驟 2 / 共 3 步** — 下一步：產出報告

## 目標

逐項驗證每份文件的完整性與品質，識別缺口與風險。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. PRD 驗證

檢查 PRD 是否包含：

- [ ] 專案概述（類型、領域、複雜度）
- [ ] 功能需求（含優先級）
- [ ] 用戶故事
- [ ] 驗收標準（每個功能）
- [ ] 非功能性需求（效能、安全、可擴展性）
- [ ] 邊界條件與排除項目
- [ ] 風險與緩解策略
- [ ] 所有區段已填寫（無空白 placeholder）

### 2. UX 設計驗證

檢查 UX 設計是否包含：

- [ ] 用戶旅程地圖
- [ ] 資訊架構
- [ ] 核心流程圖
- [ ] UI 元件規格
- [ ] 互動模式定義
- [ ] 響應式設計策略
- [ ] 無障礙標準

### 3. 架構文件驗證

檢查架構文件是否包含：

- [ ] 系統邊界定義
- [ ] 元件分解（含職責與依賴）
- [ ] 通訊協議定義
- [ ] 資料流設計
- [ ] 資料模型
- [ ] ADR（架構決策記錄）
- [ ] 基礎設施設計
- [ ] 部署策略
- [ ] 安全策略
- [ ] 效能策略

### 4. Epic/Story 驗證

檢查 Epic/Story 是否符合：

- [ ] 每個 Epic 有明確的目標和範圍
- [ ] 每個 Story 滿足 INVEST 原則
- [ ] 驗收標準使用 Given-When-Then 格式
- [ ] Story 覆蓋 PRD 所有功能需求
- [ ] 依賴關係已標記
- [ ] 工作量已估算

### 5. 跨文件一致性

- [ ] PRD 需求 ↔ Story 的覆蓋率 = 100%
- [ ] 架構元件 ↔ Story 涉及的元件一致
- [ ] UX 流程 ↔ Story 的功能一致
- [ ] 非功能性需求 ↔ 架構策略一致

### 6. 記錄驗證結果

為每個檢查項標記：✅ 通過 / ⚠️ 部分通過 / ❌ 未通過，並附上說明。

## 完成條件

- ✅ 所有文件的所有區段已逐項檢查
- ✅ 跨文件一致性已驗證
- ✅ 驗證結果已記錄

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-03-report.md`
