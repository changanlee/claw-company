# 步驟 4：基礎設施與部署策略

**進度：步驟 4 / 共 5 步** — 下一步：最終產出

## 目標

設計基礎設施架構、部署策略、安全措施與效能優化策略。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 基礎設施設計

- **運算資源**：伺服器/容器/serverless 的選擇與規格
- **儲存資源**：資料庫、物件存儲、CDN
- **網路架構**：VPC、子網路、負載均衡
- **環境規劃**：開發 / 測試 / 預演 / 正式環境

### 2. 部署策略

- **CI/CD 管線**：建置、測試、部署的自動化流程
- **部署方式**：Blue-Green / Canary / Rolling update
- **版本管理**：版本號規則、回滾策略
- **環境變數管理**：機密管理、設定檔分離

### 3. 安全策略

- **認證與授權**：OAuth2 / JWT / RBAC 設計
- **資料安全**：傳輸加密（TLS）、靜態加密
- **API 安全**：速率限制、輸入驗證、CORS
- **安全監控**：入侵偵測、異常行為警報
- **備份策略**：備份頻率、保留期限、恢復測試

### 4. 效能策略

- **快取策略**：CDN / Redis / 應用層快取
- **資料庫優化**：讀寫分離、連接池、查詢優化
- **前端效能**：程式碼分割、懶載入、資源壓縮
- **監控與告警**：效能指標、閾值、告警通道

### 5. 成本估算

- 基礎設施月費預估
- 隨規模增長的成本曲線
- 成本優化建議
