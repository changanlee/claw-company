# 步驟 2：系統架構設計

**進度：步驟 2 / 共 5 步** — 下一步：架構決策記錄

## 目標

設計系統架構，包含元件分解、通訊協議與資料流。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令（按順序，不可跳步）

### 1. 系統邊界定義

- 系統與外部的邊界
- 外部依賴（第三方服務、API、資料源）
- 使用者接入點

### 2. 元件分解

將系統分解為高內聚、低耦合的元件：

- **前端層**：UI 元件、狀態管理、路由
- **API 層**：端點設計、認證中間件、速率限制
- **業務邏輯層**：核心業務服務、領域模型
- **資料層**：資料庫、快取、檔案存儲
- **基礎設施層**：日誌、監控、佇列

每個元件需定義：職責、輸入/輸出、依賴關係。

### 3. 通訊協議

定義元件間的通訊方式：

- **同步通訊**：REST / GraphQL / gRPC
- **非同步通訊**：Message Queue / Event Bus / WebSocket
- **資料格式**：JSON / Protocol Buffers / 其他
- **錯誤處理**：重試策略、超時設定、斷路器

### 4. 資料流設計

繪製核心資料流（Mermaid 或 ASCII）：

- 使用者請求的完整資料路徑
- 資料寫入與讀取路徑
- 快取策略與失效機制
- 資料一致性策略

### 5. 資料模型

設計核心資料模型：

- 實體關係圖（ER diagram）
- 核心資料表結構
- 索引策略
- 資料遷移策略
