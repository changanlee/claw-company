# 步驟 1：準備開發環境

**進度：步驟 1 / 共 4 步** — 下一步：TDD 實作

## 目標

讀取 quick-spec 或直接指令，確認開發環境就緒，驗證任務適合快速開發流程。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 讀取規格

讀取 CTO spawn 時提供的 quick-spec 文件路徑或直接指令，理解：

- 功能需求和解決方案
- 實作步驟
- 測試案例和驗收標準

### 2. 確認 Git Branch

- 檢查當前 git branch
- 如需要，建立 feature branch
- 確認 branch 命名符合團隊慣例

### 3. 確認影響範圍和測試策略

- 確認需修改的檔案清單
- 確認測試策略

### 4. 範圍檢查

**如超出「小型獨立功能」範圍** → 立即停止，回報 CTO 建議走完整流程。

判斷標準：
- 修改超過 5 個檔案
- 需要修改公共 API 介面
- 涉及多個模組的交互修改
- 需要資料庫 schema 變更

## 完成條件

- ✅ quick-spec 或直接指令已讀取
- ✅ Git branch 已確認或建立
- ✅ 影響範圍和測試策略已確認
- ✅ 範圍檢查通過（適合快速開發）

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-02-implement.md`
