# 步驟 1：收集審查資料

**進度：步驟 1 / 共 3 步** — 下一步：逐項審查

## 目標

收集所有審查所需的資料，建立審查上下文。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 載入 Story File

讀取 CTO 提供的 story file，獲取：

- Story 描述與驗收標準
- 任務清單與完成狀態
- 開發備註

### 2. 收集變更檔案清單

確認需要審查的代碼變更：

- 從 CTO 提供的資訊中獲取變更檔案清單
- 或透過 git diff 獲取變更內容
- 記錄每個檔案的變更類型（新增/修改/刪除）

### 3. 載入相關規格文件

如有以下文件，一併載入：

- PRD（產品需求文件）
- 架構設計文件
- 技術規格

### 4. 建立審查清單

整理審查上下文：

- 變更範圍摘要
- 涉及的模組/元件
- 預期的行為變化

## 完成條件

- ✅ Story file 已載入
- ✅ 變更檔案清單已確認
- ✅ 相關規格文件已載入（如有）
- ✅ 審查上下文已建立

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-02-review.md`
