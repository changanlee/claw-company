# 步驟 3：確認就緒

**進度：步驟 3 / 共 3 步** — 最後一步

## 目標

確認 story file 所有欄位填寫完整，可交付開發工程師，更新 sprint-status。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 完整性檢查

逐項確認 story file 的所有必填欄位：

- [ ] Story 標題已填寫
- [ ] Story 描述已填寫
- [ ] 驗收標準至少 2 條
- [ ] 任務清單至少 1 項
- [ ] 開發備註已填寫（可為「無特殊備註」）

### 2. 品質檢查

確認內容品質：

- 驗收標準是否具體可測試（非模糊描述）
- 任務粒度是否適中（過大需再拆）
- 描述是否有歧義

### 3. 更新 Sprint Status

在 sprint-status 文件中：

- 將此 story 的狀態從 `backlog` 更新為 `ready`
- 記錄 story file 路徑

### 4. 產出摘要

整理 story 建立摘要回報 CTO：

- Story 標題與 ID
- 驗收標準數量
- 任務數量
- Story file 路徑

## 完成條件

- ✅ 所有必填欄位已確認完整
- ✅ 內容品質檢查通過
- ✅ Sprint status 已更新
- ✅ 摘要已產出

## 下一步

此為最後一步。將摘要回報 CTO，Story 已準備就緒可供開發。
