# 步驟 3：最終驗證與交付

**進度：步驟 3 / 共 3 步** — 最後一步

## 目標

執行全面驗證，確認所有驗收標準滿足，更新狀態並回報 CTO。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 執行驗證流程

讀取並遵循 `{{INSTALL_DIR}}/workspace-cto/rules/verification.md` 驗證流程。

### 2. 全套測試

執行所有層級的測試：

- **單元測試**：確認所有單元測試通過
- **整合測試**：確認模組間互動正確
- **回歸測試**：確認未破壞既有功能

### 3. 代碼品質檢查

- 執行 Linting，確認無錯誤
- 檢查代碼風格一致性

### 4. 驗收標準確認

逐項對照 story file 的驗收標準：

- 每個標準必須有對應的測試或驗證方式
- 記錄每個標準的驗證結果

### 5. 品質檢查清單

執行 `checklist.md` 品質檢查清單，逐項確認。

### 6. 更新狀態

- 更新 story file 狀態：`in-progress` → `review`
- 如有 sprint-status：更新對應狀態

### 7. 產出開發摘要

產出開發摘要回報 CTO，包含：

- 完成的任務列表
- 測試結果摘要
- 檔案變更清單
- 遇到的問題與解決方式
- 需要 CTO 注意的事項

## 完成條件

- ✅ 驗證流程已執行
- ✅ 所有測試通過（單元 + 整合 + 回歸）
- ✅ Lint 無錯誤
- ✅ 驗收標準逐項確認通過
- ✅ 品質檢查清單全部通過
- ✅ Story 狀態更新為 `review`
- ✅ 開發摘要已產出

## 下一步

此為最後一步。將開發摘要回報 CTO，等待 Code Review。
