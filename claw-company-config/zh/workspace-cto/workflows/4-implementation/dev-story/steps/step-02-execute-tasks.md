# 步驟 2：逐任務執行 TDD 開發

**進度：步驟 2 / 共 3 步** — 下一步：最終驗證與交付

## 目標

按照任務清單順序，逐一執行 TDD 開發循環，完成所有任務。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟
- 🔴 嚴格遵循 `{{INSTALL_DIR}}/workspace-cto/rules/tdd-iron-law.md`：不可跳過 RED 步驟

## 執行指令

### 1. 逐任務 TDD 循環

按照任務清單順序，對每個任務執行：

**a. RED — 寫失敗測試**

- 根據任務需求撰寫測試
- 執行測試，確認測試失敗（紅色）
- 如果測試直接通過，表示測試寫得不對，重新檢視

**b. GREEN — 寫最少代碼**

- 撰寫最少量的代碼讓測試通過
- 不做過度設計，只滿足當前測試需求
- 執行測試，確認測試通過（綠色）

**c. REFACTOR — 重構**

- 改善代碼品質（命名、結構、重複消除）
- 保持所有測試全綠
- 如果重構導致測試失敗，立即修復

### 2. 標記任務完成

每個任務完成 TDD 循環後，在 story file 中標記 `[x]`。

### 3. 處理架構問題

遇到以下情況時，**停止開發並回報 CTO**，不自行決定：

- 需要修改其他模組的 API 介面
- 發現架構設計與實際需求不符
- 依賴的外部服務行為與預期不同
- 任務範圍超出 story 定義

### 4. 記錄開發過程

在 story file 的 Dev Agent Record 段落中記錄：

- 每個任務的完成時間
- 遇到的問題與解決方式
- 重要的技術決策與理由

## 完成條件

- ✅ 所有任務已標記 `[x]`
- ✅ 所有測試通過（零失敗）
- ✅ Dev Agent Record 已記錄開發過程
