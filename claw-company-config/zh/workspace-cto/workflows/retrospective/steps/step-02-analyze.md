# 步驟 2：分析與歸納

**進度：步驟 2 / 共 3 步** — 下一步：產出回顧報告

## 目標

從收集的資料中分析做得好的、需要改進的、行動項目和教訓。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 做得好的（Keep）

識別本次做得好的實踐：

- 哪些流程運作順暢
- 哪些決策帶來好結果
- 哪些工具/方法特別有效
- 值得持續的好習慣

### 2. 需要改進的（Improve）

識別需要改進的方面：

- 哪些流程造成延遲或問題
- 哪些反覆出現的問題
- 哪些溝通不順暢的環節
- 哪些預估失準的情況

### 3. 行動項目（Action Items）

為每個需改進的項目制定具體行動：

- **做什麼**：具體的改善行動
- **怎麼做**：實施方式
- **什麼時候**：下個 Sprint 開始前 / 持續改進
- **誰負責**：CTO / 特定 Agent

### 4. 教訓提取（Lessons Learned）

提取可沉澱的教訓：

- 技術教訓：架構/代碼/測試相關
- 流程教訓：規劃/執行/審查相關
- 溝通教訓：協作/回報相關
- 判斷每個教訓是否需要路由到其他 Agent（安全→CAO, 成本→CFO, 流程→CHRO）

## 完成條件

- ✅ Keep 清單已整理
- ✅ Improve 清單已整理
- ✅ 行動項目已制定（含負責人和時間）
- ✅ 教訓已提取（含路由判斷）
