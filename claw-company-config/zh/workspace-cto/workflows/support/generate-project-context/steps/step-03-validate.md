# 步驟 3：驗證準確性

**進度：步驟 3 / 共 3 步** — 最後一步

## 目標

驗證生成的 project-context.md 準確且完整，確保 AI Agent 可以依賴其內容。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. 準確性驗證

逐項檢查 project-context.md 的內容：

- 技術棧版本是否與設定檔一致
- 目錄結構描述是否與實際一致
- 開發指令是否可執行
- 架構描述是否準確

### 2. 完整性驗證

確認沒有遺漏重要資訊：

- 所有主要模組都有提及
- 關鍵依賴都有列出
- 開發流程的關鍵步驟都有覆蓋
- 常見陷阱都有提醒

### 3. 可用性驗證

模擬 AI Agent 視角：

- 讀完後是否能理解專案全貌
- 是否知道如何開始開發
- 是否知道該遵循什麼慣例
- 是否了解重要的限制和注意事項

### 4. 產出摘要

整理摘要回報 CTO：

- project-context.md 的位置
- 涵蓋的範圍
- 驗證結果
- 建議的更新頻率

## 完成條件

- ✅ 準確性已驗證
- ✅ 完整性已驗證
- ✅ 可用性已驗證
- ✅ 摘要已產出

## 下一步

此為最後一步。將摘要回報 CTO。
