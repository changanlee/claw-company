# 步驟 2：撰寫文件

**進度：步驟 2 / 共 3 步** — 下一步：驗證文件準確性

## 目標

按照撰寫計畫的優先順序，逐一撰寫或更新技術文件。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步、不合併步驟

## 執行指令

### 1. README 撰寫（如需要）

包含以下段落：

- 專案名稱與簡介
- 快速開始（安裝、設定、運行）
- 主要功能列表
- 技術棧
- 目錄結構說明
- 貢獻指南（如適用）
- 授權資訊

### 2. API 文件撰寫（如需要）

- 端點列表（路徑、方法、描述）
- 請求/回應格式（含範例）
- 認證方式
- 錯誤碼說明
- 使用範例

### 3. 開發指南撰寫（如需要）

- 開發環境設定
- 代碼風格與慣例
- 測試策略與執行方式
- 分支策略
- 常見問題

### 4. 其他文件

按撰寫計畫完成其餘文件。

### 5. 寫入位置

- 文件寫入專案的適當位置（如 `docs/`、專案根目錄）
- 遵循專案既有的文件組織方式

## 完成條件

- ✅ 撰寫計畫中的文件已全部完成
- ✅ 文件格式一致
- ✅ 文件已寫入適當位置

## 下一步

確認完成條件滿足後，讀取並遵循：`./step-03-verify.md`
