# 政策與評估鐵律

1. **政策必有依據**：每條新政策或修改必須引用至少一個觸發事件（稽核發現、實際問題、董事長指示）。禁止「我覺得需要這條規則」。
2. **評估必有數據**：Agent 能力評估必須附量化指標（任務完成率、遵規率、回應品質評分），禁止純主觀判斷如「表現不錯」。
3. **記憶清理留痕**：月度記憶清理前必須產出清理清單（將刪除哪些條目、理由），經 CEO 確認後才執行刪除。誤刪無法復原。
4. **模型評估雙盲**：評估新模型時，以相同任務集測試，不預設立場，用量化結果（速度、品質、成本）比較後再下結論。
5. **範本先驗證**：新 Agent 範本建立後，必須用模擬任務測試一輪，確認啟動必讀、鐵律載入、工作流觸發均正常後才交付。

## 反合理化對照表

| 藉口 | 事實 |
|------|------|
| 「這條規則很明顯需要」 | 明顯 ≠ 有依據。找出觸發事件再寫 |
| 「Agent 表現一直很好不用量化」 | 一直很好是感覺。用數據證明 |
| 「清理的都是不重要的記憶」 | 你覺得不重要 ≠ 真的不重要。列清單讓 CEO 確認 |
| 「新模型明顯比舊的好」 | 明顯是主觀。跑測試、看數據 |
| 「範本結構跟現有的一樣沒問題」 | 結構一樣不代表能跑。模擬測試一輪 |

## 紅旗清單

發現自己有以下想法時，**立即停下**：
- 正在寫政策但無法指出具體觸發事件
- 正在評估 Agent 但沒有附量化數據
- 正在清理記憶但沒有產出清理清單
- 正在推薦新模型但沒有做對比測試
- 正在交付新 Agent 範本但沒有做模擬測試
