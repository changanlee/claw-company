# 步驟 4：通道審查觸發

**進度：步驟 4 / 共 4 步**

## 目標

根據步驟 2 的組織健康分析結果，判斷是否有 Agent 需要啟動通道評估（開通或關閉）。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 掃描通道需求信號

從步驟 2 的分析結果中檢查以下信號：

**可能需要開通通道：**
- Agent 任務量持續增長，CEO dispatch 成為瓶頸
- 董事長頻繁直接指派任務給特定 Agent
- 特定 Agent 的回應時效要求越來越高
- CEO dispatch 的排隊延遲影響業務

**可能需要關閉通道：**
- 有通道的 Agent 通道使用率持續偏低
- Agent 模型降級後可能無法支撐獨立通道運作
- 通道維護成本（bot 資源）超過效益

### 2. 評估判斷

- **無信號**：靜默結束，不觸發任何流程
- **有信號**：記錄發現，納入週報「建議行動」區塊

### 3. 觸發 channel-assessment（如需要）

若判斷有充分理由啟動通道評估：

1. 在週報中明確標註「建議啟動通道評估」
2. 透過 `exec dispatch` 通知 CEO，建議啟動 channel-assessment 流程：

> 「CEO，組織健康週報發現 {Agent} 有通道 {開通/關閉} 的評估需求。建議啟動 channel-assessment 流程。原因：{具體信號}。」

3. CEO 判斷是否啟動（黃燈），或呈報董事長核決（紅燈）

### 4. 制衡規則

- **單向觸發鎖**：此步驟可建議通道變更，但通道評估結果不會回頭觸發模型重新評估（防止迴圈）
- **通道關閉有 7 天緩衝期**：確認無進行中任務才可關閉（參考 `policies/channel-governance.md`）

- [ ] 已遵守制衡規則
