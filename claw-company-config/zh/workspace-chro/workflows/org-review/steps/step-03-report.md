# 步驟 3：產出週報

**進度：步驟 3 / 共 3 步**

## 目標

產出結構化的組織健康週報，送交 CEO 納入晨間簡報。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 週報結構

```markdown
# 組織健康週報 — {日期}

## 總覽
- 整體健康度：良好 / 注意 / 警告
- 本週重點：{1-2 句摘要}

## 各 Agent 狀態
| Agent | 負載 | 表現 | 記憶用量 | 備註 |
|-------|------|------|---------|------|

## 關鍵指標
- 負載均衡度：{評分}
- 協調順暢度：{評分}
- 能力覆蓋度：{評分}

## 趨勢
{與上週對比的關鍵變化}

## 建議行動
1. {優先級最高的建議}
2. {次要建議}

## 需要關注
{若有異常或風險，標註於此}
```

### 2. 智慧靜默判斷

- **正常**：週報送交 CEO，不特別通知董事長
- **注意**：週報送交 CEO，建議納入晨間簡報
- **警告**：週報送交 CEO，建議立即通知董事長

### 3. 發送週報

> **注意**：此流程由 cron 觸發，`sessions_send` 不可用（v2026.3.8 cron tight isolation）。週報由 cron delivery announce 機制自動推送到通道。

- 週報作為最終輸出，由 cron announce 推送到通道（CEO 可見）
- 同時將週報存入 `output/reports/org-health-weekly-YYYY-MM-DD.md`

### 4. 更新記錄

- 更新 MEMORY.md 中的組織健康趨勢摘要
- 將完整週報存入 memory/

## 完成標準

- [ ] 週報已產出
- [ ] 已送交 CEO
- [ ] 記錄已更新
