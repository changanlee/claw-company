# 步驟 1：確認需求

**進度：步驟 1 / 共 5 步**

## 目標

確認政策需求的類型與觸發原因，定義政策範圍。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 確認需求類型

- **新增**：目前無對應政策，需要建立
- **修改**：現有政策需要更新（因環境變化、執行問題等）
- **刪除**：現有政策已不適用

### 2. 明確觸發原因

記錄觸發政策需求的事件：

- 哪個 Agent 或事件觸發了此需求
- 目前缺乏政策造成的具體問題
- 影響範圍（單一 Agent / 多個 Agent / 全公司）

### 3. 定義政策範圍

- 政策名稱（暫定）
- 適用對象（哪些 Agent）
- 預期效果（解決什麼問題）
- 與現有政策的關係（是否取代或補充）

### 4. 檢查現有政策

讀取 `{{INSTALL_DIR}}/shared/policies/` 目錄，確認：

- 是否已有相關政策
- 是否與現有政策衝突
- 是否可合併到現有政策中

## 完成標準

- [ ] 需求類型已確認
- [ ] 觸發原因已記錄
- [ ] 政策範圍已定義
- [ ] 已檢查現有政策無衝突
