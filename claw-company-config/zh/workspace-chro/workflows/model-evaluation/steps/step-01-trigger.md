# 步驟 1：偵測觸發

**進度：步驟 1 / 共 4 步**

## 目標

偵測模型升降級的觸發條件，確認是否需要啟動評估流程。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 檢查升級觸發條件

以下任一條件達標即觸發升級評估：

- **失敗率 > 20%**：Agent 任務失敗或需重做的比例超過 20%
- **能力瓶頸**：Agent 評估報告中模型匹配度為「偏低」
- **新模型發布**：市場上有更適合的新模型

### 2. 檢查降級觸發條件

以下任一條件達標即觸發降級評估：

- **失敗率 < 5%**：Agent 表現穩定，可能使用了超規格模型
- **成本偏高**：Token 審計顯示某 Agent 成本顯著高於同級
- **任務簡化**：Agent 職責調整後，不再需要高能力模型

### 3. 外部觸發

- **新模型發布**：評估新模型是否更適合現有 Agent
- **模型退役**：供應商宣布模型即將下線
- **定價變動**：模型價格調整影響成本效益

### 4. 記錄觸發情境

| 觸發類型 | 受影響 Agent | 當前模型 | 觸發指標 | 數據 |
|---------|------------|---------|---------|------|
| 升級/降級/外部 | | smart/fast | | |
