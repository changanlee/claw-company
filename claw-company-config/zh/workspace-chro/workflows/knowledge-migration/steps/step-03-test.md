# 步驟 3：適應期測試

**進度：步驟 3 / 共 4 步**

## 目標

使用歷史任務測試新模型的回應品質，確保切換不影響 Agent 表現。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 選擇測試任務

從 Agent 的 memory/ 目錄中選取 3-5 個歷史任務，涵蓋：

- **常規任務**：Agent 最常處理的任務類型
- **複雜任務**：需要較高推理能力的任務
- **邊界任務**：曾經出過問題或需要特殊處理的任務

### 2. 執行測試

對每個測試任務：

- 模擬相同的輸入條件
- 記錄新模型的回應
- 與歷史回應（若有）對比品質

### 3. 評估維度

| 維度 | 評分（1-5） | 與舊模型對比 | 備註 |
|------|-----------|------------|------|
| 指令遵循度 | | 優/同/劣 | |
| 回應品質 | | 優/同/劣 | |
| 推理能力 | | 優/同/劣 | |
| 角色一致性 | | 優/同/劣 | |
| 回應速度 | | 優/同/劣 | |

### 4. SOUL.md 適配檢查

- 新模型是否能正確表現 SOUL.md 定義的人格特質
- 行為邊界是否被正確遵守
- 記憶規則是否被正確執行
- 若有偏差，記錄需要調整的 SOUL.md 段落

### 5. 記錄測試結果

- 測試通過的任務
- 測試不通過的任務（標記原因）
- 整體品質判斷：達標 / 需調整 / 不達標

## 完成標準

- [ ] 3-5 個歷史任務已測試
- [ ] 各維度已評分
- [ ] SOUL.md 適配已檢查
- [ ] 測試結果已記錄
