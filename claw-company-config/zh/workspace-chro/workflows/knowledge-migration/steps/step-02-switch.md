# 步驟 2：切換模型

**進度：步驟 2 / 共 4 步**

## 目標

執行模型切換，確保 Agent workspace 正常載入。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 修改模型配置

更新 `{{INSTALL_DIR}}/openclaw.json` 中對應 Agent 的模型配置：

- 找到受影響 Agent 的 `model` 欄位
- 將模型別名或模型 ID 更新為目標模型
- 確認語法正確

### 2. 重新載入 Workspace

- 通知受影響 Agent 重新載入（或等待下次心跳自動載入）
- 確認 Agent 成功使用新模型回應

### 3. 基礎功能驗證

快速檢查切換後的基礎功能：

- Agent 能否正常回應
- MEMORY.md 能否正常讀取
- 心跳能否正常執行
- `exec dispatch` 能否正常發送

### 4. 記錄切換狀態

| Agent | 原模型 | 新模型 | 切換時間 | 基礎驗證 |
|-------|--------|--------|---------|---------|
| | | | | 通過/失敗 |

若基礎驗證失敗，立即回滾至原模型。

## 完成標準

- [ ] 模型配置已更新
- [ ] Agent 已重新載入
- [ ] 基礎功能驗證通過
- [ ] 切換狀態已記錄
