# 步驟 2：評估健康度

**進度：步驟 2 / 共 3 步**

## 目標

評估各 Agent 記憶系統的健康度。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 過期條目檢查

逐一檢查各 Agent MEMORY.md 中的條目：

- **過期**：超過 30 天未被引用或更新的條目
- **冗餘**：與其他條目重複或已被取代的內容
- **過時**：記錄的事實已經改變但未更新

計算過期條目比例：過期條目數 / 總條目數

### 2. 容量健康評估

| 使用率 | 狀態 | 建議 |
|--------|------|------|
| < 50% | 健康 | 無需處理 |
| 50-80% | 注意 | 開始規劃清理 |
| 80-95% | 警告 | 需優先清理 |
| > 95% | 危險 | 立即清理 |

### 3. 結構清晰度評估

檢查 MEMORY.md 的組織結構：

- 是否有清晰的分類標題
- 條目是否按優先級或時間排列
- 是否有孤立條目（無上下文）
- 格式是否一致

評分標準（1-5 分）：
- 5：結構清晰、分類完整、無冗餘
- 4：結構尚可、少量改善空間
- 3：需要整理、有明顯冗餘
- 2：結構混亂、大量過期內容
- 1：急需全面重組

### 4. **冷層信噪比評估**

- autoRecall 命中率（recall 結果是否被 Agent 後續對話引用或使用）
- 冷層記憶增長速度（每月新增條目數）
- 雜訊指標：日常操作類記憶佔比（越高 = 雜訊越多）
- **VP 啟動信號**：命中率連續 2 個月 < 70% → 在報告中標記為「建議評估 VP 拆分」

### 5. 彙整健康度評估

| Agent | 過期比例 | 容量狀態 | 結構評分 | 整體健康 |
|-------|---------|---------|---------|---------|
| CEO | % | 健康/注意/警告/危險 | /5 | |
| ... | | | | |
