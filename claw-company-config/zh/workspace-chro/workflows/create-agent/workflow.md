# CHRO 新增 Agent 流程

## 概述

CHRO（Harper）確認新增角色的必要性，設計角色規格，使用模板建立完整的 6 檔規格包，經 CEO 審核與 CAO 合規檢查後，送董事長核決。

## 前置條件

- 新增角色的需求信號（任務持續增加、現有 Agent 過載等）
- Token 消耗數據（CFO 提供）
- 組織健康報告

## 步驟總覽

| 步驟 | 名稱 | 說明 | 核決 |
|------|------|------|------|
| 01 | 確認必要性 | 偵測信號，論證新增角色的必要性 | — |
| 02 | 設計規格 | 設計職責、模型、成本預估 | — |
| 03 | 建立規格包 | 使用模板建立 6 檔規格包 | — |
| 04 | 三方審核 | CEO 審核 + CAO 合規檢查 | 黃燈（CEO） |
| 05 | 送核決 | 送董事長核決 | 紅燈（董事長） |

## 核決閘門

| 決策項目 | 燈號 | 處理方式 |
|---------|------|---------|
| 規格包審核 | 黃燈 | 送 CEO 審核 + CAO 合規檢查 |
| 新增 Agent 生效 | 紅燈 | 經 CEO 呈報董事長核決 |
