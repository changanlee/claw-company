# 步驟 5：送核決

**進度：步驟 5 / 共 5 步**

## 目標

將經三方審核的新 Agent 規格包送董事長核決。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 準備核決文件

整理呈報文件：

- 必要性論證摘要
- 角色規格總覽（職責、模型、成本）
- 6 檔規格包
- CEO 審核意見
- CAO 合規檢查結果
- 對現有 Agent 的影響說明

### 2. 經 CEO 呈報董事長（紅燈）

呈報格式：

> **新增 Agent 核決請求**
>
> - 角色名稱：{名稱}
> - 職責摘要：{1-2 句}
> - 模型級別：smart / fast
> - 預估月成本：${金額}
> - CEO 意見：通過 / 有條件通過
> - CAO 意見：合規 / 有條件合規
> - 附件：完整規格包

### 3. 處理核決結果

- **核准**：
  1. 在 `{{INSTALL_DIR}}/` 建立新的 workspace 目錄
  2. 放入 6 檔規格包
  3. 更新 `openclaw.json` 新增 Agent 配置
  4. 更新受影響 Agent 的 AGENTS.md（職責移交）
  5. 更新 `shared/company-rules.md` 中的組織架構
  6. 執行 `openclaw agents add` 註冊新 Agent
- **有條件核准**：依董事長要求修改後執行上述步驟
- **駁回**：記錄原因，歸檔至 memory/

### 4. 更新記錄

- 更新 MEMORY.md 記錄新增 Agent 決策
- 通知所有 Agent 組織架構已變更
