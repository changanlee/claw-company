# 步驟 1：確認必要性

**進度：步驟 1 / 共 5 步**

## 目標

偵測新增角色的信號，論證其必要性。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 偵測觸發信號

檢查以下指標：

- **任務累積**：某類任務持續增加，但無專責 Agent 處理
- **Agent 過載**：某 Agent 的任務量或 Token 消耗持續偏高
- **職責缺口**：組織健康報告中反覆出現的能力真空
- **效率瓶頸**：某類任務處理時間持續超標
- **董事長指示**：董事長明確要求新增某類功能

### 2. 數據支持

收集量化數據：

- 相關任務的週/月數量趨勢
- 目前由哪些 Agent 兼任處理（及其負擔）
- Token 消耗數據（向 CFO 索取）
- 相關任務的平均完成時間

### 3. 必要性論證

回答以下問題：

1. **現有 Agent 能否吸收？** — 透過職責調整解決
2. **能否透過工作流程優化解決？** — 不需要新 Agent
3. **新增 Agent 的預期效益？** — 量化改善幅度
4. **成本影響？** — 新增 Agent 的月 Token 預算估計

### 4. 結論

- **建議新增**：信號明確、數據支持、現有方案無法解決
- **建議暫緩**：信號尚弱、可先嘗試其他方案
- **不建議**：現有架構可滿足需求

若結論為「不建議」或「暫緩」，記錄原因後結束流程。
