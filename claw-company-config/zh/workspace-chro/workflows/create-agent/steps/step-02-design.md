# 步驟 2：設計規格

**進度：步驟 2 / 共 6 步**

## 目標

設計新 Agent 的角色規格，包含職責、模型、成本預估和職責來源。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 角色定位

- **角色名稱**：中文名 + 英文縮寫
- **匯報對象**：直屬上級（CEO / 獨立）
- **在組織中的位置**：參照 company-rules.md 的組織架構
- **與現有 Agent 的關係**：協作方式、資訊流

### 2. 職責設計

- **核心職責**：3-5 項主要工作
- **職責來源**：從哪些現有 Agent 分出
  - 被分出職責的 Agent 需要更新 AGENTS.md
  - 確認分出後不影響原 Agent 的核心功能
- **職責邊界**：明確「不做什麼」
- **心跳邏輯**：定期巡檢做什麼

### 3. 模型選擇

- **建議模型級別**：smart / fast
- **選擇理由**：基於任務複雜度和品質要求
- **預估月 Token 消耗**：參照類似 Agent 的消耗量

### 4. 成本預估

| 項目 | 估計 |
|------|------|
| 月 Token 消耗 | |
| 月成本（基於模型定價） | |
| 占總預算比例 | |
| 預期效益（節省時間/品質提升） | |

### 5. 通道配置

- 是否需要獨立 Telegram Bot（如 CAO）
- 還是共用 CEO 通道
- Cron 排程需求

## 完成標準

- [ ] 角色定位已明確
- [ ] 職責已設計並確認來源
- [ ] 模型已選擇並說明理由
- [ ] 成本已預估
- [ ] 通道配置已規劃
