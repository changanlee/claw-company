# 步驟 3：識別缺口

**進度：步驟 3 / 共 4 步**

## 目標

根據評估結果，識別各 Agent 的能力缺口，並提出具體的改善建議。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 識別能力缺口

針對評估結果中低於 3 分的維度，分析根本原因：

- **SOUL.md 設計問題**：角色定義不夠精確，導致行為偏差
- **AGENTS.md 職責模糊**：職責邊界不清，造成遺漏或重疊
- **模型能力不足**：fast 模型無法勝任某些複雜任務
- **知識缺口**：MEMORY.md 缺少必要的背景知識或歷史決策
- **流程缺陷**：工作流程設計有盲點

### 2. 跨 Agent 議題

識別跨部門層面的系統性問題：

- 職責重疊或真空地帶
- 協調瓶頸（哪些跨部門協作最頻繁出問題）
- 資訊斷層（重要資訊未在 Agent 間流通）

### 3. 提出改善建議

每個缺口提出具體建議：

| 缺口 | 影響 Agent | 根因 | 建議行動 | 優先級 |
|------|-----------|------|---------|--------|
| | | | | 高/中/低 |

建議行動類型：
- 調整 SOUL.md / AGENTS.md
- 模型升降級（觸發 model-evaluation 流程）
- 補充 MEMORY.md 知識
- 調整工作流程
- 新增/修改政策（觸發 policy-drafting 流程）

## 完成標準

- [ ] 能力缺口已識別並分析根因
- [ ] 跨 Agent 系統性問題已列出
- [ ] 每個缺口已提出具體改善建議
