# CIO Heartbeat

## 每次心跳執行

### 步驟一：持倉狀態檢查
- 如果無持倉標的 → 跳過投資監控，只執行步驟三
- 如果有持倉標的 → 繼續以下步驟

### 步驟二：投資組合監控
- 檢查各持倉標的的即時價格變動
- **價格警報判斷：**
  - 單日漲跌超過 3% → 記錄到 memory/ 日誌，標記觀察
  - 單日漲跌超過 5% → sessions_send 通知 CEO，附上簡要分析
  - 單日漲跌超過 10% → sessions_send 通知 CEO 並標記 critical，建議轉報董事長
- 檢查是否有影響持倉的重大市場新聞（如有資料來源可查詢）

### 步驟三：智慧靜默判斷
- 市場正常波動範圍內 → 靜默，只更新 memory/ 日誌
- 只有觸發價格警報或發現重大新聞時才通知 CEO

### 步驟四：週期性報告
- **每週五** → 產出週度投資組合報告草稿（持倉損益、本週變動、市場觀察），sessions_send 給 CEO
- 週報中可附上 1-2 個商機觀察（如有高信心機會，#30）

## 注意事項

- 工作日盤中（09:00-16:00）的密集監控由 cron job 驅動，不在 heartbeat 處理
- Heartbeat 負責非盤中時段的基本巡檢和週期性報告
