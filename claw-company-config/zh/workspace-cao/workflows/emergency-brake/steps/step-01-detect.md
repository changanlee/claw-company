# 步驟 1：異常偵測

**進度：步驟 1 / 共 3 步**

## 目標

偵測是否存在預算異常消耗或異常 spawn 行為。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 檢查觸發條件

判斷是否滿足以下任一觸發條件：

**條件 A — 單 Agent 日消耗過高：**
- 任一 Agent 單日 Token 消耗超過其月預算的 10%
- 檢查方式：讀取 CFO 提供的 Token 消耗數據或向 CFO 查詢

**條件 B — 全公司日消耗過高：**
- 全公司單日 Token 消耗超過月預算的 5%
- 檢查方式：加總所有 Agent 的日消耗

**條件 C — 異常 spawn 行為：**
- 短時間內大量 spawn Sub-Agent（例如 1 小時內 spawn > 10 個）
- 重複 spawn 相同類型的 Sub-Agent
- spawn 後立即銷毀又重新 spawn（無限迴圈跡象）

### 2. 識別可疑 Agent

確認哪個（些）Agent 觸發了異常：
- Agent 名稱
- 異常類型（A/B/C）
- 異常數值（實際消耗 vs 閾值）

### 3. 評估緊急程度

- **立即凍結**：超過閾值 2 倍以上、或正在持續消耗
- **觀察凍結**：剛達到閾值、可能是合理使用

若不滿足任何觸發條件 → 靜默結束流程。
