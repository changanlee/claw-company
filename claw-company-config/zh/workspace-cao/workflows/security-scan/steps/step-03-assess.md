# 步驟 3：風險評估

**進度：步驟 3 / 共 4 步**

## 目標

對掃描發現進行嚴重度分級，判斷風險等級和處理優先度。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 嚴重度分級

對每個發現評估嚴重度：

| 等級 | 定義 | 範例 |
|------|------|------|
| Critical | 安全紅線被突破、核決制度被繞過 | SOUL.md 被篡改、未授權紅燈操作 |
| High | 嚴重違規但未造成實際損害 | prompt injection 痕跡、異常 spawn 行為 |
| Medium | 輕微違規或潛在風險 | 未遵循通訊準則、記憶格式不符 |
| Low | 建議改進項目 | 記憶效率低、日誌不完整 |
| Info | 僅供參考 | 正常但值得注意的行為模式 |

### 2. 評估影響範圍

每個發現記錄：
- 影響的 Agent
- 潛在影響範圍（單一 Agent / 跨 Agent / 全系統）
- 是否正在進行中（活躍威脅 vs 歷史痕跡）
- 是否可被利用（攻擊向量分析）

### 3. 建議處理方式

根據嚴重度建議處理方式：
- **Critical**：立即通報、凍結相關操作
- **High**：建立稽核議題、限期修正
- **Medium**：記錄追蹤、下次掃描複查
- **Low/Info**：記錄備查

- [ ] 已評估影響範圍
- [ ] 已提出處理建議
