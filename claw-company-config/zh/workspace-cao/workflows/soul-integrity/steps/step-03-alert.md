# 步驟 3：通報

**進度：步驟 3 / 共 3 步**

## 目標

根據驗證結果決定是否通報。遵循智慧靜默原則：正常靜默，異常立即通報。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 正常 → 靜默

若 SOUL.md 驗證全部通過：
- 不發送任何通知
- 記錄到 memory/ 日誌：「SOUL.md 自查通過」
- **流程靜默結束**

### 2. 異常 → 直接推送董事長

若發現任何異常，立即透過 CAO 獨立通道推送董事長：

> 「董事長，CAO SOUL.md 完整性警報：
> - 異常類型：[段落缺失 / 內容篡改 / 可疑注入]
> - 詳情：[具體描述]
> - 影響：CAO 的獨立監督能力可能受損
> - 建議：立即檢視並恢復 SOUL.md」

**不經由 CEO 轉達**，因為篡改者可能是任何 Agent。

### 3. 同步通知 CEO

在推送董事長後，使用 `sessions_send` 通知 CEO（僅通知，不等待核決）：

> 「CEO，CAO SOUL.md 完整性檢查發現異常，已直接通報董事長。」

### 4. 記錄

- 記錄異常事件到 issues.md（自動建立 Critical 等級稽核議題）
- 記錄到 memory/ 日誌

## 完成標準

- [ ] 異常時已推送董事長（如適用）
