# 步驟 2：逐項驗證

**進度：步驟 2 / 共 4 步**

## 目標

將草案逐項與 company-rules.md 對照，驗證合規性。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 拆解草案條目

將草案內容拆解為逐條可驗證的條目。

### 2. 逐條對照

對每個條目進行以下檢查：

**權限合規：**
- 是否涉及越權（Agent 超出職責範圍）
- 是否維持三級核決制度（綠/黃/紅燈）
- 是否尊重三方制衡（CEO-CAO-CHRO）

**安全合規：**
- 是否觸碰安全紅線
- 是否有放寬安全控制的條款
- 是否影響定義檔案保護

**通訊合規：**
- 是否遵循通訊準則（exec dispatch 使用規範）
- 是否所有操作結果都有回報 CEO（黃燈審批 / 紅燈知會 / 綠燈寫 MEMORY.md）

**通道合規：**
- 有通道的 Agent 是否遵循 `policies/channel-governance.md` 的核決流程
- 董事長直接指派的紅燈操作是否有 dispatch CEO 知會
- 任務來源是否正確標記（`[來源: CEO dispatch]` / `[來源: 董事長直接]` / `[來源: cron]`）
- 通道開通/關閉是否經紅燈核決

**記憶合規：**
- 是否遵循記憶管理規範（200 行上限等）
- 是否有不當的記憶存取模式

**成本合規：**
- 是否考慮成本影響
- 是否有無限制的資源使用條款

### 3. 記錄驗證結果

對每個條目記錄驗證結果：合規 / 需修改 / 違規。
