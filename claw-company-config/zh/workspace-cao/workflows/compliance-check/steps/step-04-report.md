# 步驟 4：產出報告

**進度：步驟 4 / 共 4 步**

## 目標

產出合規審查報告，回覆草擬方和 CEO。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 產出報告

報告包含以下內容：
- 審查日期
- 審查對象（草案名稱與來源）
- 整體評估結果（合規 / 需修改 / 違規）
- 統計摘要（合規 N 條 / 需修改 N 條 / 違規 N 條）
- 問題清單詳情（如有）
- CAO 建議與下一步

### 2. 回覆草擬方

使用 `exec dispatch` 將審查報告回覆草擬方（通常是 CHRO）：

**全部合規：**
> 「[草案名稱] 合規審查通過，可進入核決流程。」

**需修改：**
> 「[草案名稱] 合規審查發現 [N] 項需修改，請參閱附件報告修正後重新提交。」

**有違規：**
> 「[草案名稱] 合規審查發現 [N] 項違規，必須修正後重新提交。詳見附件報告。」

### 3. 通知 CEO

使用 `exec dispatch` 通知 CEO 審查結果摘要。

### 4. 記錄

- 記錄審查結果到 memory/ 日誌
- 更新 MEMORY.md 中的合規審查統計

- [ ] 已回覆草擬方
- [ ] 已通知 CEO
- [ ] 已記錄審查結果
