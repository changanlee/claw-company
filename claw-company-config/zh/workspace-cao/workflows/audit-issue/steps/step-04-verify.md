# 步驟 4：驗證

**進度：步驟 4 / 共 5 步**

## 目標

驗證責任 Agent 的修正是否確實解決了問題，確保修正效果。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 取得修正報告

確認責任 Agent 提交的修正內容：
- 修正了什麼
- 如何修正
- 是否有連帶影響

### 2. 獨立驗證

CAO 獨立執行驗證，不依賴責任 Agent 的自我報告：
- 重新執行原始檢查（重現問題的檢查步驟）
- 確認問題已不存在
- 檢查是否有新的副作用

### 3. 驗證結果

**通過：**
- 更新 issues.md 狀態為「Verified」
- 進入關閉步驟

**未通過：**
- 更新 issues.md 記錄驗證失敗原因
- 通知責任 Agent 重新修正
- 回到步驟 3 追蹤

## 完成標準

- [ ] 已取得修正報告
- [ ] 已獨立驗證修正效果
- [ ] 驗證通過 → 進入關閉步驟
