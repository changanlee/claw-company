# 核決權限矩陣

## 三級核決制度

### 綠燈 — 自動執行（不需核決）

| 操作類型 | 說明 |
|---------|------|
| 資料收集 | 網路搜尋、市場資料抓取、新聞整理 |
| 內部記錄 | 記帳、行程登記、會議記錄、日誌寫入 |
| 內部通訊 | Agent 間的 sessions_send |
| 例行心跳 | heartbeat 巡檢、狀態回報 |
| 分析產出 | 報告草稿、建議草稿（不對外發送） |
| Agent 命名 | 董事長為 Agent 取名，更新 SOUL.md 名字欄位 |

### 黃燈 — 需 CEO 審批

| 操作類型 | 說明 |
|---------|------|
| 消費建議 | 任何消費方案（CEO 審後轉呈董事長） |
| 投資建議 | 買入/賣出/調倉建議 |
| 行程規劃 | 出行方案草稿 |
| 開發方案 | 技術方案、架構決策 |
| 政策草案 | CHRO 草擬的新政策或修改 |

### 紅燈 — 需董事長核決

| 操作類型 | 金額/條件 |
|---------|----------|
| 任何花費 | > $50 USD（或等值） |
| 對外通訊 | 替董事長發送任何對外訊息 |
| 訂票付款 | 所有訂票與付款操作 |
| 程式碼推送 | 推送到 main branch |
| 政策生效 | 任何 policies/ 變更正式生效 |
| SOUL.md 修改 | 任何 Agent 的 SOUL.md 變更 |
| HEARTBEAT.md 修改 | 任何 Agent 的心跳邏輯變更（頻率、觸發條件、巡視清單） |
| IDENTITY.md 修改 | 任何 Agent 的身份資訊變更（命名除外，命名為綠燈） |
| AGENTS.md 修改 | 任何 Agent 的職責與工作流程變更 |
| 工程師定義修改 | engineers/*.md 角色定義或 rules/*.md 開發紀律變更 |
| 安全事件 | CAO 標記為嚴重的安全事件 |

## 緊急例外

如果情況緊急且無法即時聯繫董事長（等待超過 30 分鐘無回應）：
- CEO 可暫時核准黃燈事項，事後補報
- 紅燈事項無例外，必須等待董事長核決
