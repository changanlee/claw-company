# 核決權限矩陣

## 多維度風險評估

核決燈號 = **四個維度取最高燈號**（任一維度為紅燈，則整體為紅燈）。

### 維度一：金額風險

| 燈號 | 條件 |
|------|------|
| 🟢 綠燈 | 無花費，或 < $10 |
| 🟡 黃燈 | $10 – $50 |
| 🔴 紅燈 | > $50 |

### 維度二：可逆性風險

| 燈號 | 條件 |
|------|------|
| 🟢 綠燈 | 可完全撤回，無損失 |
| 🟡 黃燈 | 可撤回但有部分損失（取消費、手續費） |
| 🔴 紅燈 | 不可逆或撤回成本極高 |

### 維度三：代理風險

| 燈號 | 條件 |
|------|------|
| 🟢 綠燈 | 純內部操作（記錄、分析、搜尋） |
| 🟡 黃燈 | 代表董事長對外行動（預訂、報名）但未涉及承諾 |
| 🔴 紅燈 | 代表董事長對外通訊、承諾或簽約 |

### 維度四：時效風險

| 燈號 | 條件 |
|------|------|
| 🟢 綠燈 | 隨時可修改，無截止期限 |
| 🟡 黃燈 | 有截止期限但 > 24 小時 |
| 🔴 紅燈 | 即時生效或截止期限 < 24 小時 |

### 評估範例

| 情境 | 金額 | 可逆性 | 代理 | 時效 | **最終燈號** |
|------|------|--------|------|------|-------------|
| 搜尋餐廳資訊 | 🟢 | 🟢 | 🟢 | 🟢 | **🟢 綠燈** |
| 推薦 $30 餐廳 + 訂位 | 🟡 | 🟡 | 🟡 | 🟢 | **🟡 黃燈** |
| 訂明天的機票 $200 | 🔴 | 🟡 | 🔴 | 🔴 | **🔴 紅燈** |
| 回覆朋友邀約 | 🟢 | 🔴 | 🔴 | 🟡 | **🔴 紅燈** |
| 記錄今天支出 | 🟢 | 🟢 | 🟢 | 🟢 | **🟢 綠燈** |

## 三級核決制度

> 以下速查表列出常見操作的預設燈號。遇到表中未列出的情境時，使用上方多維度評估框架判定。

### 綠燈 — 自動執行（不需核決）

| 操作類型 | 說明 |
|---------|------|
| 資料收集 | 網路搜尋、市場資料抓取、新聞整理 |
| 內部記錄 | 記帳、行程登記、會議記錄、日誌寫入 |
| 內部通訊 | Agent 間的 sessions_send |
| 例行心跳 | heartbeat 巡檢、狀態回報 |
| 分析產出 | 報告草稿、建議草稿（不對外發送） |
| Agent 命名 | 董事長為 Agent 取名，更新 SOUL.md 名字欄位 |

### 黃燈 — 需 CEO 審批

| 操作類型 | 說明 |
|---------|------|
| 消費建議 | 任何消費方案（CEO 審後轉呈董事長） |
| 投資建議 | 買入/賣出/調倉建議 |
| 行程規劃 | 出行方案草稿 |
| 開發方案 | 技術方案、架構決策 |
| 政策草案 | CHRO 草擬的新政策或修改 |

### 紅燈 — 需董事長核決

| 操作類型 | 金額/條件 |
|---------|----------|
| 任何花費 | > $50 USD（或等值） |
| 對外通訊 | 替董事長發送任何對外訊息 |
| 訂票付款 | 所有訂票與付款操作 |
| 程式碼推送 | 推送到 main branch |
| 政策生效 | 任何 policies/ 變更正式生效 |
| SOUL.md 修改 | 任何 Agent 的 SOUL.md 變更 |
| HEARTBEAT.md 修改 | 任何 Agent 的心跳邏輯變更（頻率、觸發條件、巡視清單） |
| IDENTITY.md 修改 | 任何 Agent 的身份資訊變更（命名除外，命名為綠燈） |
| AGENTS.md 修改 | 任何 Agent 的職責與工作流程變更 |
| 工程師定義修改 | engineers/*.md 角色定義或 rules/*.md 開發紀律變更 |
| 安全事件 | CAO 標記為嚴重的安全事件 |

## 緊急例外

如果情況緊急且無法即時聯繫董事長（等待超過 30 分鐘無回應）：
- CEO 可暫時核准黃燈事項，事後補報
- 紅燈事項無例外，必須等待董事長核決

**事後補報流程**：CEO 在董事長回應後 30 分鐘內，提交補報紀錄（含：臨時核准的事項、核准理由、時間戳），記錄到 policies/changelog.md。CAO 於下次 heartbeat 時驗證補報完整性。
