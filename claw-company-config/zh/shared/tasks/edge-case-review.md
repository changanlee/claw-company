# 邊界條件獵人

## 目的

系統性地走過每個分支路徑和邊界條件，找出**未被處理的**邊界情況。與對抗式審查不同 — 這是方法驅動的逐條檢查，不是態度驅動的批判。

## 執行方式

1. **識別所有分支** — 列出內容中的每個決策點、條件判斷、分類。

2. **逐條檢查邊界** — 對每個分支問：
   - 輸入為空/null 時會怎樣？
   - 輸入為極大值/極小值時會怎樣？
   - 輸入格式錯誤時會怎樣？
   - 多個條件同時觸發時會怎樣？
   - 時序問題：事件順序不同時會怎樣？
   - 並發問題：多個 Agent 同時操作時會怎樣？
   - 資源問題：Token 不夠 / 檔案不存在 / 網路斷線時會怎樣？

3. **只報告未處理的** — 已有處理機制的邊界條件不需要報告。

4. **產出報告** — 格式：

```markdown
## 邊界條件審查報告

**審查對象：** {文件名或描述}
**審查者：** {Agent 名稱}

### 未處理的邊界條件

| # | 分支點 | 邊界條件 | 可能後果 | 建議處理方式 |
|---|--------|---------|---------|-------------|
| 1 | {位置} | {條件} | {後果} | {建議} |
```

## 注意事項

- 聚焦在**未處理**的邊界條件，不要報告已處理的
- 與對抗式審查正交 — 這是方法驅動的，不是態度驅動的
