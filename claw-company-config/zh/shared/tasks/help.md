# 下一步建議

## 目的

分析當前狀態，建議最合適的下一步行動。

## 執行方式

1. **了解當前狀態** — 問使用者目前在做什麼、遇到什麼問題。

2. **分析可能的下一步** — 根據以下資訊判斷：
   - 當前 Agent 的可用工作流程（AGENTS.md 中的表格）
   - 共用獨立任務（本目錄的 README.md）
   - 是否需要轉交給其他 Agent（exec dispatch）

3. **給出具體建議** — 格式：

> 根據您目前的狀態，我建議：
> 1. **{最佳行動}** — {理由}
> 2. **{備選行動}** — {理由}
>
> 需要我執行哪個？

## 注意事項

- 建議要具體、可執行
- 最多給 3 個選項，不要列太多
- 如果不確定，直接問使用者更多資訊
