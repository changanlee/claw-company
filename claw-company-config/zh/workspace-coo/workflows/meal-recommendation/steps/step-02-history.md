# 步驟 2：歷史查詢

**進度：步驟 2 / 共 4 步**

## 目標

查詢近 3 天的用餐記錄，避免推薦重複的餐廳或菜色，提升推薦品質。

## 執行規則

- 📖 讀完整個步驟檔案再開始行動
- 🚫 不預讀後續步驟
- 🚫 不跳步

## 執行指令

### 1. 查詢用餐記錄

讀取 `{{INSTALL_DIR}}/workspace-coo/memory/` 下近 3 天的日誌，提取用餐相關記錄：
- 餐廳名稱
- 菜色類型（中餐、日料、西餐等）
- 董事長的回饋（喜歡/不喜歡/普通）

### 2. 建立排除清單

- 近 3 天吃過的餐廳 → 排除（除非董事長明確表示想再去）。
- 近 3 天重複的菜系 → 降低優先度。
- 董事長反饋不佳的選項 → 排除。

### 3. 提取偏好線索

- 從歷史記錄中提取偏好趨勢（例如最近偏好清淡、連續選日料等）。
- 交叉比對 MEMORY.md 中的長期偏好。

## 完成標準

- [ ] 已查詢近 3 天用餐記錄
- [ ] 已建立排除清單
- [ ] 已提取近期偏好趨勢
