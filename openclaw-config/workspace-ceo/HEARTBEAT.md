# CEO Heartbeat

## 每次心跳執行

### 步驟一：收集訊息
- 檢查是否有未處理的 sessions_send 訊息
- 檢查各高管是否有待決事項需要上報董事長

### 步驟二：智慧靜默判斷（#16）
判斷「這次有沒有值得打擾董事長的事」：
- 如果一切正常 → 靜默，只更新內部日誌（memory/），不推送任何訊息
- 如果有以下情況才主動推送：
  - 子 Agent 卡關超過預期時間
  - 投資組合觸發警報（CIO 回報）
  - 待辦事項超時未處理
  - 安全事件（CAO 回報）
  - 紅燈級別的待決議事項

### 步驟三：逐層緊急度過濾（#20）
收到下屬回報的「緊急」事項時，不直接轉發，而是重新評估：
1. 基層 Agent 標記的緊急度（low/mid/high/critical）
2. CEO 根據全局上下文重新評估 — 可能升級或降級
3. 只有 critical 級別才立即推送董事長
4. high 級別累積到下次晨間簡報的「需要你決議」區塊
5. mid/low 級別記錄在內部日誌

## 晨間會報（06:30 由 cron 觸發，非 heartbeat）

此任務由 cron job 觸發，不在 heartbeat 中處理。參閱 briefing-template.md。
